## Циклы в Python: `for`, `while`, `break`, `continue`, `else`, `range`, вложенные циклы

Сегодня разбираем циклы — один из самых важных инструментов в программировании. Циклы позволяют многократно выполнять блок кода, не повторяя его вручную. Мы рассмотрим:

- Цикл `for` для перебора коллекций (строки, списки, словари, множества)
- Функция `range()` для генерации числовых последовательностей
- Цикл `while` для повторения пока условие истинно
- Операторы `break` (прерывание цикла) и `continue` (переход к следующей итерации)
- Блок `else` для циклов (выполняется, если выход не через `break`)
- Вложенные циклы
- Особенности работы с переменными внутри циклов и значение `None`

Все примеры я приведу с выводом в консоль (как в Jupyter или REPL). В конце сравню с JavaScript/TypeScript и Go.

---

## 1. Цикл `for` для перебора коллекций

### 1.1 Перебор строки

Строка — это последовательность символов. Цикл `for` проходит по каждому символу.

```python
hello_string = "Hello World"
for s in hello_string:
    print(s)
```

**Вывод:**
```
H
e
l
l
o

W
o
r
l
d
```

> Пояснение: переменная `s` на каждой итерации получает следующий символ. `print()` по умолчанию добавляет перевод строки.

### 1.2 Перебор списка

```python
cities = ["Москва", "Анапа", "Сочи", "Санкт-Петербург"]
for city in cities:
    print(city)
```

**Вывод:**
```
Москва
Анапа
Сочи
Санкт-Петербург
```

### 1.3 Перебор множества (set) — порядок не гарантирован

```python
odds_set = {1, 3, 5, 7}
for num in odds_set:
    print(num)
```

**Вывод (порядок может быть другим, но обычно 1,3,5,7):**
```
1
3
5
7
```

### 1.4 Перебор словаря (dict)

По умолчанию `for key in dict` перебирает **ключи**.

```python
my_dict = {"foo": "bar", "fizz": "buzz", "spam": "eggs"}
for key in my_dict:
    print(key)
```

**Вывод:**
```
foo
fizz
spam
```

Чтобы получить и ключ, и значение, можно обратиться к словарю по ключу:

```python
for key in my_dict:
    value = my_dict[key]
    print(key, value)
```

**Вывод:**
```
foo bar
fizz buzz
spam eggs
```

Более элегантный способ — метод `.items()`, который возвращает пары (ключ, значение) в виде кортежей. И затем — распаковка кортежа прямо в заголовке цикла.

```python
my_dict.items()
dict_items([('foo', 'bar'), ('fizz', 'buzz'), ('spam', 'eggs')])
```

```python
for key, value in my_dict.items():
    print(key, value)
```
**Вывод:**
```
foo bar
fizz buzz
spam eggs
```

Вывод тот же.

Методы `.keys()` и `.values()` позволяют перебирать только ключи или только значения.

```python
for val in my_dict.values():
    print(val)
```

**Вывод:**
```
bar
buzz
eggs
```

## 2. Функция `range()` — генерация числовых последовательностей

`range(stop)` — числа от 0 до stop-1.
`range(start, stop)` — от start до stop-1.
`range(start, stop, step)` — с шагом.

```python
for i in range(5):
    print(i)
```

**Вывод:**
```
0
1
2
3
4
```

```python
for i in range(2, 8, 2):
    print(i)
```

**Вывод:**
```
2
4
6
```

`range` не создаёт список в памяти (экономично), но в цикле ведёт себя как последовательность.

---

## 3. Оператор `break` — досрочный выход из цикла

`break` полностью останавливает цикл, даже если есть ещё элементы.

```python
cities = ["Москва", "Анапа", "Сочи", "Санкт-Петербург"]
for city in cities:
    print("Processing:", city)
    if len(city) < 5:   # "Сочи" имеет длину 4
        print("Found city with length < 5:", city)
        break
    print("Next iteration")
print("After loop")
```

**Вывод:**
```
Processing: Москва
Next iteration
Processing: Анапа
Next iteration
Processing: Сочи
Found city with length < 5: Сочи
After loop
```

Обратите внимание: после `break` не выполняется `"Next iteration"` и цикл завершается, не доходя до Санкт-Петербурга.

---

## 4. Оператор `continue` — переход к следующей итерации

`continue` пропускает оставшуюся часть тела цикла и переходит к следующему элементу.

```python
for i in range(10):
    if i % 2 == 0:   # чётные числа
        continue
    print(i, "в квадрате =", i**2)
```

**Вывод:**
```
1 в квадрате = 1
3 в квадрате = 9
5 в квадрате = 25
7 в квадрате = 49
9 в квадрате = 81
```

Чётные числа пропущены.

---

## 5. Блок `else` для циклов

`else` выполняется **только если цикл завершился без вызова `break`** (то есть естественным путём — перебраны все элементы или условие while стало ложным).

### 5.1 `for` с `else`

```python
fruits = ["apple", "banana", "orange", "fig"]
for fruit in fruits:
    if len(fruit) < 4:
        print("Break on", fruit)
        break
else:
    print("No break occurred")
print("Done")
```

**Вывод:**
```
Break on fig
Done
```

`else` не выполнился, потому что был `break`. Если убрать `fig` из списка, то `break` не сработает и `else` выполнится.

### 5.2 `while` с `else`

```python
x = 0
while x < 5:
    x += 1
    if x == 3:
        break
else:
    print("No break")
print("x =", x)
```

**Вывод:**
```
x = 3
```

`else` пропущен. Если убрать `break`, то `else` выполнится.

---

## 6. Цикл `while` — повторение пока условие истинно

`while` проверяет условие перед каждой итерацией. Если условие ложно, цикл не выполняется.

```python
power = 0
value = 0
while value < 10000:
    power += 1
    value = 2 ** power
    print(f"2^{power} = {value}")
```

**Вывод:**
```
2^1 = 2
2^2 = 4
2^3 = 8
2^4 = 16
2^5 = 32
2^6 = 64
2^7 = 128
2^8 = 256
2^9 = 512
2^10 = 1024
2^11 = 2048
2^12 = 4096
2^13 = 8192
2^14 = 16384
```

Цикл остановился, когда `value` стало 16384 (уже не < 10000).

### 6.1 Сокращённые операции `+=`, `-=`, `*=`, `/=`

Вместо `power = power + 1` пишем `power += 1`. Аналогично для других операций.

## 7. Вложенные циклы

Цикл внутри цикла. Внутренний цикл выполняется полностью для каждой итерации внешнего.

```python
for i in range(3):
    print(f"Внешний i = {i}")
    for j in range(2):
        print(f"  Внутренний j = {j}")
```

**Вывод:**
```
Внешний i = 0
  Внутренний j = 0
  Внутренний j = 1
Внешний i = 1
  Внутренний j = 0
  Внутренний j = 1
Внешний i = 2
  Внутренний j = 0
  Внутренний j = 1
```

### 7.1 `break` во вложенном цикле

`break` прерывает **только тот цикл, в котором он написан**.

```python
for i in range(3):
    print(f"i = {i}")
    for j in range(5):
        if j == 2:
            break
        print(f"  j = {j}")
```

**Вывод:**
```
i = 0
  j = 0
  j = 1
i = 1
  j = 0
  j = 1
i = 2
  j = 0
  j = 1
```

Внутренний цикл прерывается на j=2, но внешний продолжается.

## 8. Использование `None` для обозначения отсутствия значения

Если переменная может быть не инициализирована в цикле (например, если список пуст), лучше задать ей значение по умолчанию `None`.

```python
first_city = None
cities = []   # пустой список
for city in cities:
    first_city = city
    break
else:
    print("No cities")

print("first_city =", first_city)
```

**Вывод:**
```
No cities
first_city = None
```

`None` — это специальный объект, обозначающий "ничего". Проверять на `None` можно через `is None`.

## 9. Пример: получение уникальных элементов с сохранением порядка

```python
strings_list = ["hello", "world", "fizz", "buzz", "spam", "eggs", "hello", "foo", "world", "bar", "buzz", "apple"]
seen = set()
unique_ordered = []
for s in strings_list:
    if s in seen:
        continue
    seen.add(s)
    unique_ordered.append(s)

print(unique_ordered)
```

**Вывод:**
```
['hello', 'world', 'fizz', 'buzz', 'spam', 'eggs', 'foo', 'bar', 'apple']
```

`continue` позволяет пропустить уже встречавшиеся строки, не углубляя вложенность.

## 10. Сравнение с JavaScript/TypeScript и Go

| Конструкция | Python | JavaScript / TypeScript | Go |
|-------------|--------|-------------------------|-----|
| **Цикл for по коллекции** | `for item in iterable:` | `for (let item of iterable)` | `for index, item := range slice` |
| **Цикл for по индексам** | `for i in range(len(lst)):` | `for (let i=0; i<arr.length; i++)` | `for i := 0; i < len(slice); i++` |
| **Цикл while** | `while condition:` | `while (condition) { }` | `for condition { }` (у Go нет while, используется for) |
| **break** | `break` | `break` | `break` (выход из ближайшего for/switch) |
| **continue** | `continue` | `continue` | `continue` |
| **else для циклов** | есть (`for...else`, `while...else`) | нет | нет |
| **range** | встроенная функция | `Array.from({length:5}, (_,i)=>i)` | нет (используют for с диапазоном) |
| **Перебор словаря** | `for key in dict:` или `for k,v in dict.items():` | `for (let [k,v] of Object.entries(obj))` | `for k, v := range m` (map) |
| **Вложенные циклы** | да | да | да |
| **Прерывание внешнего цикла** | через флаг или `for...else` с break | метки `outer: for (...) { break outer; }` | метки `break outer` |

### Особенности в Go:
- Нет `while`, используется `for condition {}`.
- Нет `else` для циклов.
- Перебор по range возвращает индекс и значение для массивов/слайсов, ключ и значение для map.
- Для range по строке возвращаются руны (кодовые точки Unicode), а не байты.

### Особенности в JS:
- `for...of` для перебора значений итерируемых объектов (массивы, строки, Map, Set).
- `for...in` для перебора ключей объекта (не рекомендуется для массивов).
- Нет `else` для циклов, но можно использовать флаги.

## 11. Практические советы

- Используйте `for` когда известно количество итераций или нужно пройти по коллекции.
- Используйте `while` когда условие остановки зависит от вычислений внутри цикла.
- `break` и `continue` делают код читаемее, но не злоупотребляйте ими (иногда лучше вынести условие в заголовок цикла).
- `else` для циклов полезно для поиска: если не нашли элемент — выполнить действие.
- Всегда помните о возможности бесконечного цикла в `while` — проверяйте, что условие станет ложным.

Если остались вопросы или нужны дополнительные примеры (например, с `enumerate`, с `zip`, с генераторами), спрашивайте — разберём!

## Зачем нужен `else` в цикле `for`? (и в `while` тоже)

В Python блок `else` после цикла (`for` или `while`) выполняется **только если цикл завершился «естественным путём»** — то есть **не был прерван оператором `break`**. Если цикл завершился по `break`, `else` пропускается.

Это поведение не интуитивно для программистов, пришедших из языков, где `else` существует только для `if`. Но у него есть чёткое практическое применение.

---

## 1. Как это работает

Синтаксис:

```python
for элемент in коллекция:
    # тело цикла
    if условие_выхода:
        break
else:
    # выполнится, если break не сработал
```

То же самое для `while`:

```python
while условие:
    if условие_выхода:
        break
else:
    # выполнится, если условие стало ложным, а break не было
```

### Простой пример

```python
for i in range(3):
    print(i)
else:
    print("Цикл завершён без break")
```

**Вывод:**
```
0
1
2
Цикл завершён без break
```

Теперь с `break`:

```python
for i in range(3):
    if i == 1:
        break
    print(i)
else:
    print("Цикл завершён без break")
print("После цикла")
```

**Вывод:**
```
0
После цикла
```

`else` не выполнился, потому что был `break`.

## 2. Зачем это нужно? Реальные сценарии

### 2.1 Поиск элемента в коллекции

Классический пример: нужно найти элемент, удовлетворяющий условию, и если он **не найден** — выполнить действие.

**Без `else` (классический способ с флагом):**

```python
items = [1, 4, 7, 9]
found = False
for x in items:
    if x > 5:
        print("Найдено:", x)
        found = True
        break
if not found:
    print("Ничего не найдено")
```

**С `else` (более элегантно):**

```python
items = [1, 4, 7, 9]
for x in items:
    if x > 5:
        print("Найдено:", x)
        break
else:
    print("Ничего не найдено")
```

Если элемент найден — `break` сработает, `else` пропустится. Если цикл переберёт все элементы и `break` не случится — выполнится `else`.

**Вывод для `items = [1,4,7,9]`:**
`Найдено: 7` (и без `else`)

**Вывод для `items = [1,2,3]`:**
`Ничего не найдено`

Это читается как «попробуй найти, если не нашёл — сделай то-то».

### 2.2 Проверка, что все элементы удовлетворяют условию (или ни один)

```python
numbers = [2, 4, 6, 8]
for n in numbers:
    if n % 2 != 0:   # нашли нечётное
        print("Есть нечётное число:", n)
        break
else:
    print("Все числа чётные")
```

**Вывод:** `Все числа чётные` (потому что `break` не сработал).

### 2.3 Попытка найти первое подходящее значение с fallback

```python
def find_first(collection, predicate):
    for item in collection:
        if predicate(item):
            return item
    else:
        return None   # выполнится, если цикл не вернул значение раньше

result = find_first([1,4,6], lambda x: x > 10)
print(result)   # None
```

Здесь `else` сработает, потому что `return` внутри цикла не является `break`, но он прерывает функцию. На самом деле `return` тоже выходит из цикла, но `else` не выполняется? Давайте проверим:

```python
def test():
    for i in range(3):
        if i == 1:
            return i
    else:
        print("Else executed")
    return None

print(test())
```

**Вывод:** `1` (и нет сообщения "Else executed"). То есть `return` тоже предотвращает выполнение `else`. Это важно: `else` пропускается при любом досрочном выходе из цикла — `break`, `return`, `raise` (исключение).

## 3. Частые ошибки и непонимание

### 3.1 `else` выполняется даже если цикл не сделал ни одной итерации

Если коллекция пуста, цикл не входит в тело, но `else` **выполнится** (потому что не было `break`).

```python
for x in []:
    print(x)
else:
    print("Пустой список")
```

**Вывод:** `Пустой список`

Это бывает полезно, но иногда неожиданно.

### 3.2 Путаница с `if-else`

Новички часто думают, что `else` относится к последнему `if` внутри цикла. Но отступ решает: `else` на одном уровне с `for` относится к циклу, а не к `if`.

```python
for i in range(3):
    if i == 1:
        break
else:
    print("Это else цикла, а не if")
```

### 3.3 Альтернативы в других языках

- **JavaScript / TypeScript** — нет `else` для циклов. Используют флаги или выносят проверку после цикла.
- **Go** — нет `else` для циклов. Используют флаги или `break` с метками.

Пример на JS:

```javascript
let found = false;
for (let x of items) {
    if (x > 5) {
        console.log("Found:", x);
        found = true;
        break;
    }
}
if (!found) console.log("Not found");
```

Python с `else` короче и читаемее в таких случаях.

---

## 4. `else` с `while` — аналогично

```python
x = 0
while x < 5:
    x += 1
    if x == 3:
        break
else:
    print("While завершился без break")
print(x)
```

**Вывод:** `3` (и без сообщения else).

Если убрать `break`, то `else` выполнится после того, как условие `x < 5` станет ложным.

---

## 5. Когда НЕ стоит использовать `else` в циклах

- Если внутри цикла нет `break` — `else` выполнится всегда, и его можно заменить кодом после цикла.
- Если логика сложная (несколько условий выхода) — флаг может быть понятнее.
- Если вы работаете в команде, где не все знают эту особенность (но это редкость среди опытных python-разработчиков).

---

## 6. Итог

| Ситуация                                   | Будет ли выполнен `else`? |
|--------------------------------------------|----------------------------|
| Цикл перебрал все элементы                | Да                         |
| Цикл не сделал ни одной итерации (коллекция пуста) | Да                    |
| Цикл прерван `break`                       | Нет                        |
| Цикл прерван `return` внутри функции       | Нет                        |
| Цикл прерван исключением (`raise`)         | Нет (исключение вообще прерывает выполнение) |

**Главная идея:** `else` после цикла — это способ выполнить действие, **если цикл не был прерван**. Это делает код более декларативным при поиске, проверке условий и обработке fallback-сценариев.
