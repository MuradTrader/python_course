### 2. Числовые типы данных: `int` и `float`

В Python числа бывают:
- **Целые (`int`)** — без дробной части.
- **С плавающей точкой (`float`)** — с дробной частью (внутри хранятся как двоичные дроби, возможны微小误差).

#### Арифметические операции

| Оператор | Значение               | Пример (ввод) | Вывод |
|----------|------------------------|---------------|--------|
| `+`      | сложение               | `1 + 2`       | `3`    |
| `-`      | вычитание              | `5 - 3`       | `2`    |
| `*`      | умножение              | `5 * 6`       | `30`   |
| `/`      | обычное деление (всегда `float`) | `10 / 2`   | `5.0`  |
| `//`     | целочисленное деление (пол) | `10 // 4` | `2`    |
| `%`      | остаток от деления     | `10 % 4`      | `2`    |
| `**`     | возведение в степень   | `2 ** 3`      | `8`    |

**Почему `10 / 2` даёт `5.0`, а не `5`?**
Python гарантирует, что результат `/` — всегда `float`, даже если деление целое. Это называется *консистентность* (всегда один тип).

```python
# Примеры в Jupyter / REPL
10 / 2
```
Вывод: `5.0`

```python
10 / 4
```
Вывод: `2.5`

```python
10 // 4   # целочисленное деление (отбрасывает остаток)
```
Вывод: `2`

```python
10 % 4    # остаток
```
Вывод: `2`

```python
9 // 10   # меньше делителя → 0
```
Вывод: `0`

```python
9 % 10    # остаток = само число
```
Вывод: `9`

```python
2 ** 3    # 2 в степени 3
```
Вывод: `8`

**Работа с разными типами в одном выражении:**
Если складывать `int` и `float`, Python автоматически приводит `int` к `float` (неявное приведение).

```python
1.5 + 2     # 1.5 (float) + 2 (int) → float
```
Вывод: `3.5`

**Большие числа:** Python поддерживает длинную арифметику (ограничено только памятью). Например, `9 ** 25` выдаст огромное целое число без переполнения.

```python
2 ** 64
```
Вывод: `18446744073709551616`

**Сравнение с другими языками:**

| Операция | Python | JavaScript | TypeScript | Go |
|----------|--------|------------|------------|-----|
| Целочисленное деление | `//` | `Math.floor(a/b)` или `~~(a/b)` (для положительных) | то же, что JS | `a / b` (если оба `int`, результат `int`; иначе `float`) |
| Остаток | `%` (всегда знак делителя) | `%` (знак делимого) | то же | `%` (знак делимого) |
| Возведение в степень | `**` | `**` (ES7) или `Math.pow` | `**` | `math.Pow(a, b)` (float64) |
| Деление `/` | всегда float | всегда float | всегда float | если оба int → int (целочисленное), иначе float |

**Важно в Go:** `10 / 4` даст `2` (int), а `10.0 / 4` даст `2.5`.

### 3. Строки (`str`)

Строки создаются с помощью одинарных `' '` или двойных `" "` кавычек. Для многострочных — тройные кавычки `'''` или `"""`.

#### Конкатенация (сложение строк)

```python
"foo" + "bar"
```
Вывод: `'foobar'`

```python
"spam " + "and " + "ex"
```
Вывод: `'spam and ex'`

#### Умножение строки на число

```python
"foo" * 5
```
Вывод: `'foofoofoofoofoo'`

#### Индексация (доступ по индексу)

Индексы начинаются с **0**. Последний символ — индекс `len(строка) - 1`.

```python
s = "Hello World!"
s[0]   # 'H'
s[4]   # 'o'
s[11]  # '!'
```

```python
len(s)   # 12
```

#### Срезы (slicing)

Синтаксис: `строка[начало:конец]` — включает `начало`, не включает `конец`.
Можно опустить начало (начало с 0) или конец (до конца строки).

```python
s = "Hello World!"
s[3:8]   # с индекса 3 по 7
```
Вывод: `'lo Wo'` (индексы: 3=L,4=o,5= ,6=W,7=o)

```python
s[3:]    # с 3 до конца
```
Вывод: `'lo World!'`

```python
s[:5]    # с начала до индекса 5 (не включая 5)
```
Вывод: `'Hello'`

```python
s[:]     # полная копия
```
Вывод: `'Hello World!'`

**Отрицательные индексы** (с конца):
`s[-1]` — последний символ (`!`), `s[-2]` — предпоследний (`d`).

#### Экранирование и спецсимволы

`\n` — новая строка, `\'` — одинарная кавычка внутри строки в одинарных кавычках и т.д.

```python
print("foo\nbar")
```
Вывод:
```
foo
bar
```

#### Многострочные строки (тройные кавычки)

```python
long_text = """Hello line 1
Hello line 2

Hello line 4"""
print(long_text)
```
Вывод:
```
Hello line 1
Hello line 2

Hello line 4
```

**Сравнение с JS/TS/Go:**

- **JS/TS:** строки immutable, срезы через `slice()` или `substring()`. Конкатенация через `+`. Умножение строк не поддерживается (`"foo".repeat(5)`). Многострочные — шаблонные литералы `` `...` ``.
- **Go:** строки immutable, срезы работают по байтам (не всегда по символам из-за UTF-8). Конкатенация `+` или `strings.Builder`. Умножения нет. Многострочные — обратные кавычки `` `...` ``.

### 4. Тип `bool` (логический)

Возможные значения: `True` и `False` (с заглавной буквы).

```python
True
```
Вывод: `True`

```python
False
```
Вывод: `False`

#### Сравнения

```python
"hello" == "hello"   # True
"hello" == "Hello"   # False (регистр важен)
3 == 3.0             # True (числовое равенство)
3 == "3"             # False (разные типы)
```

#### Логические операторы

- `and` — логическое И
- `or` — логическое ИЛИ
- `not` — отрицание

```python
(3 > 2) and (5 < 10)   # True
(3 > 2) or (5 > 10)    # True
not (3 > 2)            # False
```

#### `bool` как подкласс `int`

`True == 1`, `False == 0`. Можно даже складывать:

```python
True + True    # 2
True + False   # 1
False + False  # 0
```

#### Функция `type()` и `isinstance()`

```python
type(42)           # <class 'int'>
type(3.14)         # <class 'float'>
type("hello")      # <class 'str'>
type(True)         # <class 'bool'>

isinstance(42, int)    # True
isinstance(True, bool) # True
isinstance(True, int)  # True (потому что bool наследник int)
```

**Сравнение:**

- **JS/TS:** `typeof 42 === "number"`, `true + true === 2` (тоже работает). `instanceof` для классов.
- **Go:** статическая типизация, но можно использовать `reflect.TypeOf()`. Логический тип `bool`, константы `true`/`false`. `true + true` не скомпилируется.

### 5. Переменные

Имена переменных: буквы, цифры, подчёркивание, но не начинаются с цифры. Принято использовать `snake_case` (все в нижнем регистре, слова через `_`).

```python

```

Присваивание не выводит значение в Jupyter (нет `Out`). Чтобы увидеть значение, просто напишите имя переменной или используйте `print()`.

```python
hello_string          # в Jupyter покажет значение
print(hello_string)   # выведет в консоль
```
Вывод `print`: `Hello World`

```python
type(hello_string)    # <class 'str'>
```

#### Динамическая типизация

Переменная может менять тип:

```python
x = 10        # int
x = "text"    # теперь str
```

**Сравнение:**
- **JS/TS:** динамическая (JS) / статическая с аннотациями (TS). `let x = 10; x = "text"` в JS можно, в TS ошибка.
- **Go:** статическая, тип переменной фиксирован.

### 6. Преобразование типов (явное приведение)

```python
int("123")        # 123
float("123")      # 123.0
str(123)          # "123"
int(123.999)      # 123 (просто отбрасывает дробную часть, НЕ округляет)
```


### 7. Булев контекст и условные операторы (`if`, `elif`, `else`)

Любое значение в Python может быть приведено к `bool` неявно в условиях.

**Что считается `False`?**
- `None`
- `False`
- Ноль для чисел (`0`, `0.0`)
- Пустые последовательности/коллекции: `""`, `[]`, `()`, `{}`, `set()`
- Всё остальное — `True`.

```python
bool(0)       # False
bool(1)       # True
bool(-5)      # True
bool("")      # False
bool("abc")   # True
```

#### Синтаксис `if`

```python
a = 3
if a > 2:
    print("a > 2")
```
Вывод: `a > 2`

**Важно:** блок кода после `if` выделяется отступом (обычно 4 пробела). Отсутствие фигурных скобок — ключевая особенность Python.

```python
a = 3
if a > 3:
    print("a > 3")
else:
    print("a <= 3")
```
Вывод: `a <= 3`

#### `elif` (else if)

```python
a = 3
if a > 3:
    print("a > 3")
elif a == 3:
    print("a == 3")
else:
    print("a < 3")
```
Вывод: `a == 3`

#### Проверка на "пустоту"

```python
name = ""
if name:
    print("Имя не пустое")
else:
    print("Имя пустое")
```
Вывод: `Имя пустое`

#### Комбинирование условий

```python
x = 5
if x > 0 and x < 10:
    print("x от 1 до 9")
```

#### Вложенные условия

```python
value = 1.5
if isinstance(value, (int, float)):
    if int(value) == value:   # нет дробной части
        print("Целое число (или float .0)")
    else:
        print("Дробное число")
else:
    print("Не число")
```
Вывод: `Дробное число`

**Сравнение:**

- **JS/TS:** `if (condition) { ... } else { ... }` с фигурными скобками. `elif` нет, используется `else if`.
- **Go:** `if condition { ... } else { ... }`, фигурные скобки обязательны. `else if` работает. Условный оператор без круглых скобок вокруг условия.

# Мой вопрос

```
То есть когда мы выводим любое значение это class в Python ?
Как это работает ?

```
Отличный вопрос! Ты правильно заметил, что когда мы вызываем `type(42)`, Python выводит `<class 'int'>`. И это наводит на мысль: **каждое значение в Python — это объект, принадлежащий какому-то классу**. Да, действительно, **всё в Python является объектом**, и у каждого объекта есть класс (тип), который определяет, что можно делать с этим объектом.

Давай разберёмся шаг за шагом, как это работает, и я покажу тебе примеры с выводом в консоль.

---

## 1. Что такое класс в Python?

**Класс** — это «чертёж» или «шаблон» для создания объектов. Он описывает:
- какие **атрибуты** (данные) будут у объекта,
- какие **методы** (функции) можно вызывать на объекте.

Когда ты создаёшь число `42`, Python за кулисами создаёт **объект** класса `int`. Этот объект хранит значение 42 и предоставляет методы типа `bit_length()`, `to_bytes()` и т.д.

**Пример:**
```python
x = 42
print(type(x))   # <class 'int'>
```
Вывод: `<class 'int'>`

---

## 2. Как проверить, что значение — это объект?

Даже простые литералы, такие как числа, строки и булевы значения, создаются как объекты.

```python
print(type(10))        # <class 'int'>
print(type(3.14))      # <class 'float'>
print(type("hello"))   # <class 'str'>
print(type(True))      # <class 'bool'>
print(type(None))      # <class 'NoneType'>
```

Все эти `int`, `float`, `str`, `bool` — это **встроенные классы** Python.

---

## 3. Что даёт класс объекту?

Класс определяет:
- **операторы** (например, `+`, `-`, `*`, `/`),
- **методы** (например, `upper()` для строк, `bit_length()` для int),
- **атрибуты** (данные, связанные с объектом).

Пример: у строки есть метод `upper()`:
```python
s = "hello"
print(s.upper())   # HELLO
print(type(s))     # <class 'str'>
```

Число `int` не имеет метода `upper()`, но имеет метод `bit_length()`:
```python
n = 42
print(n.bit_length())   # 6 (42 в двоичной 101010, 6 бит)
```

---

## 4. Можно ли создать свой класс?

Да, с помощью ключевого слова `class`.

```python
class Dog:
    def bark(self):
        return "Woof!"

my_dog = Dog()        # создаём объект (экземпляр) класса Dog
print(type(my_dog))   # <class '__main__.Dog'>
print(my_dog.bark())  # Woof!
```

---

## 5. Как работает `type()`?

`type()` возвращает **класс** переданного объекта. Этот класс сам является объектом (экземпляром метакласса `type`). Да, классы в Python — тоже объекты!

```python
print(type(int))   # <class 'type'>
print(type(str))   # <class 'type'>
```

Так что `int` — это объект класса `type`. А `type` — это метакласс, который создаёт все остальные классы. Но это уже глубже, для начала достаточно понимать, что у каждого значения есть класс.

---

## 6. А что, если я вывожу просто значение без `type()`?

Когда ты в Jupyter или REPL пишешь `42`, Python вычисляет выражение и показывает его **строковое представление** (вызывается метод `__repr__`). Но за кулисами это всё равно объект класса `int`.

```python
42            # REPL покажет 42, но это int-объект
```

Можешь убедиться:
```python
print((42).__class__)   # <class 'int'>
```

---

## 7. Сравнение с другими языками

| Язык       | Всё ли объект? | Как узнать тип | Примечание |
|------------|----------------|----------------|-------------|
| **Python** | Да (даже классы, функции, модули) | `type(obj)`, `isinstance()` | Динамическая типизация, утиная типизация |
| **JavaScript** | Почти всё, кроме примитивов (но они оборачиваются) | `typeof`, `instanceof` | Примитивы: число, строка, boolean — не объекты, но ведут себя как объекты при вызове методов (autoboxing) |
| **TypeScript** | Как JS, но с статической аннотацией типов | `typeof`, `instanceof` (во время выполнения) | Добавляет интерфейсы, но классы — это синтаксический сахар над прототипами |
| **Go** | Нет, базовые типы (int, float64, string) — не объекты, нет методов | `reflect.TypeOf()` | Структуры могут иметь методы, но нет наследования классов (есть встраивание) |

**Главное отличие Python** — даже число `42` имеет методы (например, `(42).bit_length()`). В Go у числа `42` нет методов, это просто значение. В JS число `42` — примитив, но при вызове `(42).toString()` временно оборачивается в объект `Number`.

---

## 8. Почему это важно?

Потому что в Python **единообразие**: любой объект можно передать в функцию, сохранить в переменную, добавить в список, проверить его тип. Это даёт гибкость: ты можешь написать функцию, которая работает с любым объектом, у которого есть нужный метод (утиная типизация).

```python
def print_length(obj):
    print(len(obj))   # у любого объекта с методом __len__

print_length("hello")   # 5
print_length([1,2,3])   # 3
print_length({"a":1})   # 1
# print_length(42)      # Ошибка: у int нет __len__
```

## 9. Практический пример: исследуем классы

```python
x = 100
print(x.__class__)        # <class 'int'>
print(x.__class__.__name__)  # 'int'

# Узнаем все методы, доступные для int
print(dir(x))   # огромный список, включая __add__, __sub__, bit_length и т.д.
```

Вывод `dir(x)` покажет все методы и атрибуты объекта `x`. Многие начинаются и заканчиваются на двойное подчёркивание — это «магические методы» (например, `__add__` для `+`).

---

## 10. Итог

- **Да, любое значение в Python — это объект, и у него есть класс (тип).**
- Класс определяет, как объект ведёт себя (какие операции и методы поддерживает).
- Функция `type()` возвращает класс объекта.
- Классы сами являются объектами (экземплярами метакласса `type`).
- Это единообразие — одна из сильных сторон Python, позволяющая писать гибкий и выразительный код.
