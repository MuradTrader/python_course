## Работа с файловой системой в Python: модули `os` и `pathlib`, импорты, чтение/запись файлов

В этом занятии мы выходим за рамки чистого Python и начинаем взаимодействовать с операционной системой: читаем содержимое папок, создаём, удаляем, читаем и записываем файлы. Также научимся импортировать дополнительные модули, узнаем про конструкцию `if __name__ == "__main__"` и познакомимся с двумя подходами к работе с путями — классическим (`os.path`) и современным объектно-ориентированным (`pathlib.Path`).

Я буду показывать код так, как он выполняется в PyCharm (или любом другом терминале), с выводом в консоль. В конце сравню с JavaScript/TypeScript и Go.

## 1. Модули и импорты

В Python **модуль** — это файл с расширением `.py`. Имя файла (без `.py`) становится именем модуля. Встроенные функции (`print`, `len`, `type`, `isinstance`) доступны без импорта. Но для работы с операционной системой нужно импортировать модуль `os` (или `pathlib`).

```python
import os
```

После этого мы можем использовать имена из модуля через точку: `os.name`, `os.path.join` и т.д.

Можно импортировать только нужную часть:

```python
from pathlib import Path
```

Теперь можно писать `Path.cwd()` вместо `pathlib.Path.cwd()`.

## 2. Запуск скриптов и `if __name__ == "__main__"`

Когда Python запускает файл напрямую (например, `python demo_os.py`), специальная переменная `__name__` получает значение `"__main__"`. Если же файл импортируется как модуль в другой скрипт, `__name__` становится равным имени модуля (без расширения). Конструкция `if __name__ == "__main__":` позволяет выполнять код **только при прямом запуске**, но не при импорте. Это хороший тон для скриптов, которые могут быть и исполняемыми, и переиспользуемыми.

Пример:

```python
# demo_os.py
def main():
    print("Hello from main")

if __name__ == "__main__":
    main()
```

**Вывод при запуске:** `Hello from main`

Если бы мы импортировали этот файл из другого модуля, `main()` не вызвался бы автоматически.

## 3. Модуль `os` — работа с операционной системой

### 3.1 Имя операционной системы: `os.name`

- `'posix'` — Linux, macOS, WSL (Windows Subsystem for Linux)
- `'nt'` — Windows (native)

```python
import os

print(os.name)   # на моей системе 'posix'

is_windows = os.name == 'nt'
print("Windows?", is_windows)   # False
```

### 3.2 Переменная `__file__` и базовая директория

`__file__` — это путь к текущему файлу (скрипту). `os.path.dirname()` возвращает директорию, в которой лежит файл.

```python
print(__file__)   # /home/user/projects/demo_os.py

base_dir = os.path.dirname(__file__)
print(base_dir)   # /home/user/projects
```

**Константы (соглашение об именовании):**
Переменные, записанные заглавными буквами с подчёркиваниями (`BASE_DIR`, `IS_WINDOWS`), по договорённости считаются константами — их не следует менять.

```python
BASE_DIR = os.path.dirname(__file__)
IS_WINDOWS = os.name == 'nt'
```

### 3.3 Разделитель путей: `os.path.sep`

В разных ОС используются разные разделители:
- Windows: `\` (обратный слеш)
- Linux/macOS: `/` (прямой слеш)

```python
print(os.path.sep)   # '/'
```

В Windows получили бы `'\\'` (один обратный слеш, экранированный в строке).

### 3.4 Построение путей: `os.path.join`

**Плохие способы** (не переносимы, легко ошибиться):

```python
# Конкатенация строк
path = BASE_DIR + os.path.sep + "pictures" + os.path.sep + "cat.jpg"

# join через разделитель
path = os.path.sep.join([BASE_DIR, "pictures", "cat.jpg"])
```

**Правильный способ** — `os.path.join()`, который сам использует правильный разделитель для текущей ОС.

```python
folder_name = "pictures"
file_name = "cat.jpg"
full_path = os.path.join(BASE_DIR, folder_name, file_name)
print(full_path)   # /home/user/projects/pictures/cat.jpg
```

### 3.5 Текущая рабочая директория: `os.getcwd()`

`cwd` — current working directory. Это папка, из которой был запущен скрипт. Она может не совпадать с `BASE_DIR`, если скрипт запущен из другого места.

```python
cwd = os.getcwd()
print(cwd)   # /home/user/projects
```

### 3.6 Содержимое папки: `os.listdir()`

`os.listdir(path)` возвращает список имён файлов и папок (включая скрытые, начинающиеся с точки).

```python
items = os.listdir(".")
print(items)   # ['venv', 'demo_os.py', '.idea', ...]
```

Проверим существование папки перед чтением:

```python
if os.path.isdir(".idea"):
    print(os.listdir(".idea"))   # содержимое .idea
```

Для файлов используем `os.path.isfile()`.

### 3.7 Создание, запись и чтение файлов

#### Режимы открытия файлов

- `'w'` (write) — перезапись файла (создаёт новый или очищает существующий).
- `'a'` (append) — добавление в конец.
- `'r'` (read) — чтение (по умолчанию).

#### Контекстный менеджер `with open`

Он гарантирует, что файл будет закрыт автоматически (даже при ошибке). Без `with` нужно вручную вызывать `.close()` — легко забыть.

```python
file_name = "file.txt"

# Запись
with open(file_name, 'w') as f:
    f.write("Hello\n")
    f.write("World\n")

# Добавление
with open(file_name, 'a') as f:
    f.write("Appended line\n")

# Чтение всех строк (возвращает список с символами \n)
with open(file_name, 'r') as f:
    lines = f.readlines()
    print(lines)   # ['Hello\n', 'World\n', 'Appended line\n']

# Чтение всего содержимого как одной строки
with open(file_name, 'r') as f:
    content = f.read()
    print(content)   # Hello\nWorld\nAppended line\n
```

**Запись нескольких строк без переносов:**
`writelines()` не добавляет автоматически переводы строк, их нужно включать в сами строки.

```python
lines_with_newline = ["first\n", "second\n", "third\n"]
with open(file_name, 'w') as f:
    f.writelines(lines_with_newline)
```

### 3.8 Удаление файла: `os.unlink()` / `os.remove()`

```python
if os.path.isfile(file_name):
    os.unlink(file_name)   # или os.remove(file_name)
    print("Deleted")
```

## 4. Модуль `pathlib` — объектно-ориентированный подход

Модуль `pathlib` появился в Python 3.4 и предлагает удобные классы для работы с путями. Основной класс — `Path`. Он автоматически использует `PosixPath` или `WindowsPath` в зависимости от ОС.

### 4.1 Импорт и базовые операции

```python
from pathlib import Path

# Текущая рабочая директория
cwd = Path.cwd()
print(cwd)                     # /home/user/projects

# Домашняя директория
home = Path.home()
print(home)                    # /home/user

# Объект для текущего файла
current_file = Path(__file__)
print(current_file)            # demo_pathlib.py (относительный путь)
```

### 4.2 Методы для информации о пути

```python
print(current_file.resolve())  # абсолютный путь
print(current_file.parent)     # папка, содержащая файл
print(current_file.parents)    # генератор всех родительских папок
print(list(current_file.parents))  # [Path('/home/user/projects'), Path('/home/user'), ...]
```

### 4.3 Перебор содержимого папки: `.iterdir()`

Возвращает генератор объектов `Path` (можно преобразовать в список).

```python
for path in cwd.iterdir():
    print(path.name, "is_dir:", path.is_dir(), "is_file:", path.is_file())
```

### 4.4 Построение путей: оператор `/` и `joinpath()`

Синтаксис с косой чертой (даже на Windows) очень нагляден:

```python
folder = "pictures"
filename = "cat.jpg"
full_path = cwd / folder / filename
print(full_path)   # /home/user/projects/pictures/cat.jpg
```

Альтернатива: `joinpath()`.

```python
full_path = cwd.joinpath(folder, filename)
```

### 4.5 Свойства: суффикс, имя без расширения (stem), имя

```python
file_path = Path("cat.jpg")
print(file_path.suffix)   # .jpg
print(file_path.stem)     # cat
print(file_path.name)     # cat.jpg
```

### 4.6 Проверка существования и удаление

```python
cats_dir = Path("/cats")
print(cats_dir.exists())   # False

# Удаление файла с подавлением ошибки, если файла нет
file_path = Path("file.txt")
file_path.unlink(missing_ok=True)   # не вызовет ошибку
```

### 4.7 Чтение и запись без `with` (упрощённо)

```python
file_path = Path("file.txt")
file_path.write_text("Hello\n")      # перезапись
file_path.write_text("New content")  # снова перезапись
content = file_path.read_text()
print(content)   # New content
```

Методы `write_text()` и `read_text()` сами открывают и закрывают файл. Для больших файлов или более сложных операций всё же используют `open()` с `with`, но для простых текстовых файлов этого достаточно.

## 5. Сравнение с JavaScript/TypeScript и Go

| Действие | Python (`os` / `pathlib`) | JavaScript/Node.js | Go |
|----------|---------------------------|--------------------|-----|
| Импорт модуля | `import os`, `from pathlib import Path` | `const fs = require('fs'); const path = require('path');` | `import "os"`, `import "path/filepath"` |
| Текущая директория | `os.getcwd()` или `Path.cwd()` | `process.cwd()` | `wd, _ := os.Getwd()` |
| Содержимое папки | `os.listdir('.')` или `Path('.').iterdir()` | `fs.readdirSync('.')` | `entries, _ := os.ReadDir(".")` |
| Проверка существования | `os.path.exists()` (устарел), `os.path.isdir()`, `Path.exists()` | `fs.existsSync()` (не рекомендуется), `fs.statSync()` | `_, err := os.Stat(path)` |
| Построение путей | `os.path.join(a,b)`, `Path(a)/b` | `path.join(a,b)` | `filepath.Join(a,b)` |
| Разделитель | `os.path.sep` | `path.sep` | `string(os.PathSeparator)` |
| Создание/запись файла | `with open(f, 'w') as f: f.write(...)` | `fs.writeFileSync(f, data)` | `os.WriteFile(f, []byte(data), 0644)` |
| Чтение файла | `Path(f).read_text()` или `with open` | `fs.readFileSync(f, 'utf8')` | `data, _ := os.ReadFile(f)` |
| Удаление файла | `os.unlink(f)`, `Path(f).unlink()` | `fs.unlinkSync(f)` | `os.Remove(f)` |
| Контекстный менеджер | `with open(...) as f:` (автозакрытие) | нет встроенного, но можно через `try/finally` | нет (но `defer f.Close()` для открытых файлов) |

**Особенности:**

- **Node.js:** асинхронные версии функций (с callback или Promise) предпочтительнее синхронных (`*Sync`), но для скриптов синхронные удобны.
- **Go:** ошибки возвращаются явно, их нужно обрабатывать. Нет автоматического закрытия файлов (используют `defer`). `os.ReadDir` возвращает срез `DirEntry`, у которого есть методы `IsDir()`, `Name()`.
- **pathlib в Python** — самый высокоуровневый и читаемый способ.

## 6. Практические советы

1. **Всегда используйте `os.path.join` или `Path /`** вместо ручного сложения строк с разделителями. Это переносимо между ОС.
2. **Контекстный менеджер `with open`** обязателен при работе с файлами через классический `open()`.
3. **`pathlib.Path`** предпочтительнее для нового кода: он чище, методы названы интуитивно, легко комбинировать пути.
4. **Проверяйте существование файла/папки перед удалением** или используйте `unlink(missing_ok=True)` (только `pathlib`).
5. **Конструкция `if __name__ == "__main__"`** — стандартный способ сделать скрипт одновременно исполняемым и импортируемым.

## Функция `repr()` в Python: представление объекта для разработчика

`repr()` — это встроенная функция, которая возвращает **строковое представление объекта**, предназначенное в первую очередь для **отладки и разработки**. Её цель — дать однозначное, детализированное описание объекта, чтобы по этой строке можно было (в идеале) восстановить исходный объект или хотя бы точно понять его тип и состояние.

В отличие от `str()`, которая возвращает «красивое» представление для конечного пользователя, `repr()` возвращает «техническое» представление для программиста.

---

## 1. `repr()` vs `str()` — основное различие

| Характеристика | `repr(obj)` | `str(obj)` |
|----------------|-------------|-------------|
| Цель | Для разработчика, отладка, логи | Для пользователя, вывод в UI |
| Должна быть однозначной | Да (по возможности) | Нет |
| Может содержать кавычки, escape-последовательности | Да | Редко |
| Вызывается автоматически | В интерактивной консоли (Jupyter, REPL) | В `print()` и при форматировании строк |

**Пример для строки:**

```python
text = "Hello\nWorld"

print(repr(text))   # 'Hello\nWorld'  (с кавычками и \n)
print(str(text))    # Hello\nWorld     (без кавычек, \n становится реальным переводом строки)
```

**Вывод:**
```
'Hello\nWorld'
Hello
World
```

---

## 2. Что делает `repr(__file__)`?

`__file__` — это специальная переменная, содержащая путь к текущему скрипту (как строку). Вызов `repr(__file__)` возвращает эту строку, **обёрнутую в кавычки**, а также экранирует специальные символы (обратные слеши, кавычки, переводы строк и т.д.).

### Пример

```python
# demo.py
print(__file__)
print(repr(__file__))
```

**Вывод (на Linux/macOS):**
```
/home/user/projects/demo.py
'/home/user/projects/demo.py'
```

**Вывод (на Windows):**
```
C:\Users\user\projects\demo.py
'C:\\Users\\user\\projects\\demo.py'
```

Обратите внимание: в `repr()` обратные слеши **экранированы** (`\\`), чтобы при повторном вычислении строки они не воспринимались как начало escape-последовательности.

---

## 3. Где автоматически вызывается `repr()`?

- В интерактивной консоли (Jupyter, REPL) при вводе выражения:

```python
>>> "hello\nworld"
'hello\nworld'   # это repr, не print
```

- При выводе коллекций (список, словарь, кортеж) — их элементы отображаются через `repr()`.

```python
lst = ["a\nb", "c\td"]
print(lst)   # вызовет repr для элементов
```

**Вывод:** `['a\nb', 'c\td']`

- В отладчике (debugger) при просмотре переменных.

## 4. Как переопределить `repr()` для своих классов?

Достаточно определить метод `__repr__`. Для полноты также определяют `__str__`.

```python
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def __repr__(self):
        return f"Person('{self.name}', {self.age})"

    def __str__(self):
        return f"{self.name}, {self.age} years old"

p = Person("Alice", 30)
print(repr(p))   # Person('Alice', 30)
print(str(p))    # Alice, 30 years old
print(p)         # print вызывает str, поэтому Alice, 30 years old
```

В интерактивной консоли просто `p` показало бы `Person('Alice', 30)`.

## 5. Почему `repr()` важен при работе с путями (Path)?

Когда вы работаете с `pathlib.Path`, `repr()` явно показывает тип объекта, а не просто строку:

```python
from pathlib import Path
p = Path(__file__)

print(p)            # /home/user/projects/demo.py
print(repr(p))      # PosixPath('/home/user/projects/demo.py')   (или WindowsPath)
```

Это сразу даёт понять, что это не строка, а специальный объект. В отладке это очень удобно.

---

## 6. Когда использовать `repr()` явно?

- При логировании ошибок (чтобы видеть точные значения переменных, включая кавычки и спецсимволы).
- При сохранении данных в текстовом виде, когда нужно потом восстановить объект (например, через `eval(repr(obj))` — но с осторожностью!).
- При формировании сообщений об ошибках: `f"Invalid value: {value!r}"` (альтернативная запись через `!r`).

```python
value = "O'Reilly"
print(f"Invalid: {value}")   # Invalid: O'Reilly
print(f"Invalid: {value!r}") # Invalid: 'O\'Reilly'
```
