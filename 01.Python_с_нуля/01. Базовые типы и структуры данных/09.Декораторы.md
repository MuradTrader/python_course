## 2. Повторение: функции, аргументы и распаковка

Прежде чем перейти к декораторам, нужно вспомнить, как работают функции и аргументы.

### Простая функция без аргументов

```python
def hello():
    print("hello there")
```

Функция `hello` — это объект, который хранится в переменной `hello`. Если написать `hello` без скобок — это ссылка на функцию. Со скобками `hello()` — вызов.

```python
hello()          # вызов
print(hello)     # ссылка на объект функции
```

**Вывод:**
```
hello there
<function hello at 0x...>
```

Функция ничего не возвращает явно, значит возвращает `None`.

### Функция с аргументами (позиционные и именованные)

```python
def power_number(number, power=2):
    """Возвращает number в степени power"""
    return number ** power
```

- `number` — обязательный позиционный параметр.
- `power` — именованный параметр со значением по умолчанию.

**Примеры вызовов:**

```python
print(power_number(4, 5))       # позиционные: 4**5 = 1024
print(power_number(number=3, power=4))  # именованные: 81
print(power_number(3, power=4)) # смешанный: позиционный + именованный
```

**Вывод:**
```
1024
81
81
```

### Распаковка аргументов из коллекций

**Распаковка позиционных (`*args`)**:

```python
args_tuple = (4, 5)
print(power_number(*args_tuple))   # * распаковывает кортеж/список в позиционные аргументы
```

**Вывод:**
```
1024
```

**Распаковка именованных (`**kwargs`)**:

```python
kwargs_dict = {'number': 3, 'power': 4}
print(power_number(**kwargs_dict))   # ** распаковывает словарь в именованные аргументы
```

**Вывод:**
```
81
```

**Важно:** имена ключей в словаре должны совпадать с именами параметров функции.

### Функция с произвольным числом аргументов (`*args`, `**kwargs`)

```python
def power_numbers(*numbers, power=2):
    """Принимает любое количество чисел, возводит каждое в степень power"""
    return [n ** power for n in numbers]
```

- `*numbers` — собирает все переданные позиционные аргументы в кортеж.
- `power` — может быть передан только как именованный аргумент.

**Примеры:**

```python
print(power_numbers())                     # пустой список
print(power_numbers(2))                    # [4]
print(power_numbers(2, 3, 4))              # [4, 9, 16]
print(power_numbers(2, 3, power=3))        # [8, 27]
print(power_numbers(*range(5), power=3))   # распаковка range(5) = 0,1,2,3,4 -> [0,1,8,27,64]
```

**Вывод:**
```
[]
[4]
[4, 9, 16]
[8, 27]
[0, 1, 8, 27, 64]
```

Этот механизм (`*args`, `**kwargs`) критически важен для создания универсальных декораторов.

## 3. Что такое декоратор?

**Декоратор** — это функция, которая принимает другую функцию (оригинальную) и возвращает новую функцию (обёртку), которая обычно добавляет какое-то поведение до и/или после вызова оригинальной функции.

В Python функции — это объекты, их можно передавать как аргументы, возвращать и присваивать переменным.

### Простейший декоратор (без синтаксического сахара)

Создадим функцию `myDecorator`, которая принимает функцию и возвращает новую функцию `replacement`.

```python
def myDecorator(func_to_decorate):
    print("A: myDecorator create a new func")   # выполнится 1 раз при декорировании
    def replacement():
        print("B: running replacement function")
        func_to_decorate()                      # вызов оригинальной функции
        print("C: finished running replacement function")
    print("D: created replacement function")
    return replacement
```

Теперь у нас есть простая функция `myFunction`:

```python
def myFunction():
    print("myFunction runs")
```

Применяем декоратор вручную:

```python
myFunction = myDecorator(myFunction)
```

**Вывод после этого присваивания (выполнились print A и D):**
```
A: myDecorator create a new func
D: created replacement function
```

Теперь `myFunction` — это уже не исходная функция, а `replacement` (обёртка). Вызовем её:

```python
myFunction()
```

**Вывод:**
```
B: running replacement function
myFunction runs
C: finished running replacement function
```

Обратите внимание: print A и D были только один раз, при создании обёртки. При каждом вызове `myFunction` выполняются B, тело оригинальной функции, затем C.

### Синтаксический сахар: @

Вместо `myFunction = myDecorator(myFunction)` можно написать:

```python
@myDecorator
def small_func():
    print("I'm small func")
```

Это **точно то же самое**: Python создаёт функцию `small_func`, затем вызывает `myDecorator(small_func)` и результат присваивает имени `small_func`.

```python
@myDecorator
def small_func():
    print("I'm small func")

print("--- вызов small_func ---")
small_func()
```

**Вывод:**
```
A: myDecorator create a new func
D: created replacement function
--- вызов small_func ---
B: running replacement function
I'm small func
C: finished running replacement function
```

## 4. Проблема: потеря имени и документации

Если мы не используем специальные средства, то имя декорированной функции и её docstring подменяются на обёртку:

```python
print(small_func.__name__)   # выведет 'replacement', а не 'small_func'
```

Это неудобно для отладки. Решение — `functools.wraps`.

## 5. Использование `functools.wraps`

`wraps` — это декоратор для обёртки, который копирует метаданные (имя, docstring, аннотации) из исходной функции в обёртку.

```python
from functools import wraps

def my_deck(func):
    @wraps(func)          # важно: wraps применяется к wrapper
    def wrapper():
        print("call func")
        func()
        print("finished wrapper")
    return wrapper
```

Теперь при декорировании имя сохранится:

```python
@my_deck
def sum_func():
    print("sum func called")

print(sum_func.__name__)   # 'sum_func', а не 'wrapper'
sum_func()
```

**Вывод:**
```
sum_func
call func
sum func called
finished wrapper
```

**Как это работает?** `wraps(func)` — это декоратор, который возвращает декоратор, который обновляет атрибуты `wrapper` (например, `__name__`, `__doc__`) из `func`. Без этого имя обёртки остаётся `'wrapper'`.

## 6. Декоратор для функции с одним аргументом

Пока наши декораторы работали только с функциями без аргументов. Создадим декоратор для функции, принимающей один аргумент.

```python
def decorator_with_one_arg(func):
    @wraps(func)
    def wrapper(arg):            # принимаем один аргумент
        print(f"call {func.__name__} with arg {arg}")
        result = func(arg)       # вызываем оригинальную функцию
        print(f"result = {result}")
        return result
    return wrapper
```

Применим к функции `cube`:

```python
@decorator_with_one_arg
def cube(num):
    return num ** 3

print(cube(4))   # 64
```

**Вывод:**
```
call cube with arg 4
result = 64
64
```

## 7. Универсальный декоратор для любых аргументов

Чтобы декоратор работал с функциями, у которых разное количество позиционных и именованных аргументов, нужно использовать `*args` и `**kwargs` в обёртке.

```python
def universal_decorator(func):
    @wraps(func)
    def wrapper(*args, **kwargs):   # принимаем любые аргументы
        print(f"Calling {func.__name__} with args={args}, kwargs={kwargs}")
        result = func(*args, **kwargs)   # передаём их дальше
        print(f"Returned {result}")
        return result
    return wrapper
```

Теперь этот декоратор можно применять к любой функции:

```python
@universal_decorator
def add(a, b):
    return a + b

@universal_decorator
def greet(name, greeting="Hello"):
    return f"{greeting}, {name}!"

print(add(3, 5))
print(greet("Alice", greeting="Hi"))
```

**Вывод:**
```
Calling add with args=(3, 5), kwargs={}
Returned 8
8
Calling greet with args=('Alice',), kwargs={'greeting': 'Hi'}
Returned Hi, Alice!
Hi, Alice!
```

## 8. Полезный пример: декоратор для измерения времени выполнения

Используем `timeit.default_timer` (кросс-платформенный таймер с высоким разрешением).

```python
from timeit import default_timer
from functools import wraps

def show_timing(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        start = default_timer()
        result = func(*args, **kwargs)
        end = default_timer()
        total = end - start
        # форматируем вывод: 10 знаков после запятой для наглядности
        print(f"Executed {func.__name__} in {total:.10f} sec and got {result}")
        return result
    return wrapper
```

Проверим на функции `power_numbers` (из пункта 2):

```python
@show_timing
def power_numbers(*numbers, power=2):
    return [n ** power for n in numbers]

print(power_numbers(1, 2, 3, 4, power=3))
```

**Вывод (примерный, время зависит от компьютера):**
```
Executed power_numbers in 0.0000012000 sec and got [1, 8, 27, 64]
[1, 8, 27, 64]
```

## 9. Кэширование (мемоизация)

### Встроенный кэш: `functools.lru_cache`

`lru_cache` запоминает результаты вызовов функции для переданных аргументов. Это особенно полезно для рекурсивных функций, таких как вычисление чисел Фибоначчи.

```python
from functools import lru_cache

@show_timing
@lru_cache(maxsize=None)   # maxsize=None — без ограничения
def fib(n):
    if n < 2:
        return n
    return fib(n-1) + fib(n-2)

print(fib(10))
```

**Вывод (с измерением времени):**
```
Executed fib in 0.0000153000 sec and got 55
55
```

Без кэша рекурсивный `fib(10)` делал бы 177 вызовов, а с кэшем — только 11. Порядок декораторов важен: `@lru_cache` должен быть ближе к функции, чем `@show_timing`, чтобы кэшировался именно результат, а не измерялось время каждого кэшированного вызова. Python применяет декораторы снизу вверх: сначала `@lru_cache`, затем `@show_timing`.

### Самодельный кэш для функции с одним аргументом

Создадим декоратор, который хранит кэш в атрибуте функции.

```python
def cache_one_arg(func):
    @wraps(func)
    def wrapper(arg):
        # если ещё нет кэша, создаём пустой словарь
        if not hasattr(func, '_cache'):
            func._cache = {}
        if arg in func._cache:
            print(f"Cache hit for {arg}")
            return func._cache[arg]
        result = func(arg)
        func._cache[arg] = result
        print(f"Cache miss for {arg}, computed {result}")
        return result
    return wrapper
```

Применим к квадрату и кубу:

```python
@cache_one_arg
def square(x):
    return x * x

print(square(4))   # miss
print(square(4))   # hit
print(square(5))   # miss
print(square._cache)   # доступ к кэшу
```

**Вывод:**
```
Cache miss for 4, computed 16
16
Cache hit for 4
16
Cache miss for 5, computed 25
25
{4: 16, 5: 25}
```

Обратите внимание: кэш хранится в атрибуте функции `_cache`, и он уникален для каждой функции. Функции `square` и `cube` будут иметь разные кэши.

## 10. Порядок нескольких декораторов

Когда вы пишете:

```python
@deco1
@deco2
def func(): ...
```

Это эквивалентно `func = deco1(deco2(func))`. То есть сначала применяется `deco2` к `func`, затем к результату применяется `deco1`. **Декораторы выполняются снизу вверх**.

Пример:

```python
def deco1(f):
    print("deco1 called")
    def wrapper(*a, **kw):
        print("deco1 wrapper")
        return f(*a, **kw)
    return wrapper

def deco2(f):
    print("deco2 called")
    def wrapper(*a, **kw):
        print("deco2 wrapper")
        return f(*a, **kw)
    return wrapper

@deco1
@deco2
def hello():
    print("hello")

hello()
```

**Вывод:**
```
deco2 called
deco1 called
deco1 wrapper
deco2 wrapper
hello
```

Видно: сначала создаётся `deco2(hello)`, затем `deco1(результат)`.

## 11. Итог и практические советы

- **Декоратор** — функция, возвращающая обёртку, изменяющую поведение исходной функции.
- Всегда используйте `functools.wraps` в своих декораторах, чтобы не терять имя и docstring.
- Для универсальности обёртка должна принимать `*args, **kwargs` и передавать их дальше.
- Декораторы можно комбинировать; порядок важен.
- Встроенные полезные декораторы: `@lru_cache`, `@staticmethod`, `@classmethod`, `@property`.

## Как работают несколько декораторов: максимально подробно

Когда вы применяете к функции несколько декораторов, Python выполняет их **в определённом порядке** – как при «надевании» (декорировании), так и при последующем вызове. Давайте разберёмся с этим шаг за шагом, на очень простом примере.

### 1. Что делает один декоратор?

Напомню: декоратор – это функция, которая принимает функцию и возвращает новую функцию (обёртку). Пример:

```python
def decorator_one(func):
    print("→ Выполняется decorator_one (создание обёртки)")
    def wrapper_one():
        print("  → Начало wrapper_one")
        func()
        print("  ← Конец wrapper_one")
    return wrapper_one

@decorator_one
def hello():
    print("    Привет, мир!")

hello()
```

Вывод:
```
→ Выполняется decorator_one (создание обёртки)
  → Начало wrapper_one
    Привет, мир!
  ← Конец wrapper_one
```

Сначала `decorator_one` вызывается один раз в момент декорирования (печатает первое сообщение). При вызове `hello()` мы попадаем в `wrapper_one`.

### 2. Два декоратора: синтаксис и эквивалент

Допустим, есть два декоратора:

```python
@deco1
@deco2
def func():
    ...
```

Это **полностью эквивалентно** записи:

```python
func = deco1(deco2(func))
```

То есть **сначала применяется нижний декоратор** (`@deco2`), а затем его результат передаётся верхнему (`@deco1`). Порядок – **снизу вверх** по тексту.

### 3. Пример с двумя декораторами и печатью

Создадим два декоратора, каждый выводит сообщения при создании обёртки и при её работе.

```python
def deco1(func):
    print("1) Сработал deco1 (верхний) – начинает оборачивать то, что получил")
    def wrapper1():
        print("   >> Внутри wrapper1 (вызван первым)")
        func()   # вызов функции, которую обернули
        print("   << Выход из wrapper1")
    return wrapper1

def deco2(func):
    print("2) Сработал deco2 (нижний) – начинает оборачивать исходную функцию")
    def wrapper2():
        print("      >> Внутри wrapper2 (вызван вторым)")
        func()   # вызов исходной функции (или следующей обёртки)
        print("      << Выход из wrapper2")
    return wrapper2

@deco1   # верхний
@deco2   # нижний
def original():
    print("         ** Оригинальная функция **")
```

**Что произойдёт при выполнении этого кода?** (Ещё до вызова `original()`)

Python увидит `@deco1` и `@deco2` и начнёт обрабатывать снизу вверх:

1. Сначала обрабатывается `@deco2`: вызывается `deco2(original)`.
   - Печатается: `2) Сработал deco2 (нижний) – начинает оборачивать исходную функцию`
   - Создаётся `wrapper2`, который внутри будет вызывать `original()`.
   - `deco2` возвращает `wrapper2`.

2. Теперь результат (функция `wrapper2`) передаётся в `deco1`: вызывается `deco1(wrapper2)`.
   - Печатается: `1) Сработал deco1 (верхний) – начинает оборачивать то, что получил`
   - Создаётся `wrapper1`, который внутри будет вызывать `wrapper2`.
   - `deco1` возвращает `wrapper1`.

3. Имя `original` теперь ссылается на `wrapper1` (результат `deco1`).

**Вывод на этапе декорирования (до любого вызова функции):**
```
2) Сработал deco2 (нижний) – начинает оборачивать исходную функцию
1) Сработал deco1 (верхний) – начинает оборачивать то, что получил
```

Теперь вызовем `original()`:

```python
print("\n--- Вызов original() ---")
original()
```

**Вывод вызова:**
```
--- Вызов original() ---
   >> Внутри wrapper1 (вызван первым)
      >> Внутри wrapper2 (вызван вторым)
         ** Оригинальная функция **
      << Выход из wrapper2
   << Выход из wrapper1
```

### 4. Объяснение порядка выполнения при вызове

Схематично:

```
original()   →   вызывает wrapper1 (верхняя обёртка)
wrapper1     →   печатает ">> Внутри wrapper1"
             →   вызывает func() – а это wrapper2
wrapper2     →   печатает "   >> Внутри wrapper2"
             →   вызывает original() (исходную функцию)
original     →   печатает "      ** Оригинальная функция **"
             →   возврат в wrapper2
wrapper2     →   печатает "   << Выход из wrapper2"
             →   возврат в wrapper1
wrapper1     →   печатает "<< Выход из wrapper1"
```

Таким образом, **при вызове обёртки выполняются в порядке снаружи внутрь** – сначала код верхнего декоратора (`wrapper1`), потом код нижнего (`wrapper2`), потом исходная функция. Затем возврат идёт в обратном порядке.

### 5. Аналогия с матрёшкой

Представьте, что `original` – это самая маленькая матрёшка.
`deco2` – это средняя матрёшка, которая её оборачивает.
`deco1` – это большая матрёшка, которая оборачивает среднюю.

При декорировании вы сначала берёте маленькую матрёшку (original), кладёте её в среднюю (wrapper2), а потом среднюю кладёте в большую (wrapper1). Итог – `original` становится большой матрёшкой.

При вызове вы открываете большую матрёшку (wrapper1), внутри неё находите среднюю (wrapper2), внутри неё – маленькую (original).

### 6. Более наглядный пример с аргументами и возвратом

Декораторы часто добавляют поведение, не меняя сигнатуру. Посмотрим на декораторы, которые модифицируют результат.

```python
def uppercase(func):
    def wrapper(*args, **kwargs):
        result = func(*args, **kwargs)
        return result.upper()
    return wrapper

def exclamation(func):
    def wrapper(*args, **kwargs):
        result = func(*args, **kwargs)
        return result + "!!!"
    return wrapper

@uppercase   # верхний
@exclamation # нижний
def greet(name):
    return f"Hello, {name}"

print(greet("Alice"))
```

**Что происходит?**

1. `greet = uppercase(exclamation(greet))`
   - `exclamation(greet)` возвращает обёртку, которая добавляет `"!!!"`.
   - `uppercase(эта_обёртка)` возвращает обёртку, которая переводит результат в верхний регистр.

2. Вызов `greet("Alice")`:
   - Входим в `wrapper` от `uppercase`:
     - Вызываем `func(*args, **kwargs)`, где `func` – это обёртка от `exclamation`.
     - Входим в обёртку от `exclamation`:
       - Вызываем исходную `greet("Alice")` → получаем `"Hello, Alice"`.
       - Добавляем `"!!!"` → `"Hello, Alice!!!"`.
     - Возвращаемся в обёртку `uppercase`, применяем `.upper()` → `"HELLO, ALICE!!!"`.
   - Возвращаем результат.

**Вывод:**
```
HELLO, ALICE!!!
```

### 7. Важное замечание про `functools.wraps`

Когда вы пишете свои декораторы, всегда используйте `@wraps(func)`, чтобы не потерять имя и документацию. При нескольких декораторах `wraps` должен быть в каждом из них. Это не влияет на порядок выполнения, но сохраняет метаданные.

### 8. Резюме

| Что происходит                          | Порядок                                                  |
|-----------------------------------------|----------------------------------------------------------|
| **Декорирование** (создание обёрток)    | снизу вверх: самый нижний применяется первым            |
| **Вызов декорированной функции**        | снаружи внутрь: сначала верхний декоратор, потом нижний, затем исходная функция |

Запомните эквивалентность:
```python
@deco1
@deco2
@deco3
def f(): ...
```
равно
```python
f = deco1(deco2(deco3(f)))
```

Теперь, когда вы видите несколько `@` над функцией, вы точно знаете: сначала нижний получает исходную функцию, затем результат передаётся выше, и так до самого верхнего. При вызове всё идёт в обратном порядке, как матрёшка.
