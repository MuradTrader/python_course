## 1. Работа с обычными текстовыми файлами

### 1.1. Создание и запись файла без контекстного менеджера

```python
# Создаём переменную с именем файла (будем использовать её везде, чтобы менять только в одном месте)
FILE_PATH = "file.txt"

def main():
    # Открываем файл в режиме записи ('w' – write)
    f = open(FILE_PATH, 'w')
    # Записываем строку с символом переноса строки в конце
    f.write("hello\n")
    # Обязательно закрываем файл
    f.close()

if __name__ == "__main__":
    main()
```

**Пояснение:**
- `open(FILE_PATH, 'w')` – открывает файл для записи. Если файл не существует, он будет создан. Если существует – содержимое будет перезаписано.
- `f.write("hello\n")` – записывает строку в файл. `\n` – символ переноса строки (newline), чтобы следующая запись началась с новой строки.
- `f.close()` – закрывает файл, освобождая ресурсы. Важно не забывать закрывать, иначе данные могут не сохраниться полностью или возникнут проблемы с доступом.

**Результат:**
В текущей папке появится файл `file.txt` с содержимым:
```
hello
```

### 1.2. Запись с использованием контекстного менеджера

```python
FILE_PATH = "file.txt"

def main():
    # Контекстный менеджер with автоматически закроет файл после выхода из блока
    with open(FILE_PATH, 'w') as f:
        f.write("hello world\n")

if __name__ == "__main__":
    main()
```

**Пояснение:**
- Конструкция `with open(...) as f:` гарантирует, что после выполнения всех операций внутри блока файл будет закрыт, даже если произойдёт ошибка.
- Это предпочтительный способ, так как он сокращает код и исключает риск забыть `close()`.

**Результат:**
Файл `file.txt` перезаписывается и теперь содержит:
```
hello world
```

### 1.3. Чтение файла

```python
FILE_PATH = "file.txt"

def main():
    # Открываем файл для чтения ('r' – read, можно не указывать, т.к. это режим по умолчанию)
    with open(FILE_PATH, 'r') as f:
        # readlines() читает все строки и возвращает список строк
        lines = f.readlines()
        print(lines)
    # Файл автоматически закрыт после выхода из with

if __name__ == "__main__":
    main()
```

**Вывод в консоли:**
```
['hello world\n']
```

**Пояснение:**
- `readlines()` возвращает список строк. Каждая строка содержит символ переноса `\n` (кроме, возможно, последней, если её нет).
- Можно также использовать `read()` – прочитать весь файл как одну строку, или `readline()` – читать по одной строке.

### 1.4. Дополнение файла (append)

```python
FILE_PATH = "file.txt"

def main():
    # Режим 'a' (append) – добавляет в конец файла, не перезаписывая
    with open(FILE_PATH, 'a') as f:
        f.write("hello again\n")
        # write lines принимает список строк и записывает их
        f.writelines(["cats\n", "dogs\n"])

if __name__ == "__main__":
    main()
```

**Результат:**
После выполнения предыдущих шагов в файле `file.txt` окажется:
```
hello world
hello again
cats
dogs
```

**Пояснение:**
- Режим `'a'` открывает файл для добавления. Если файла нет, он будет создан.
- `writelines()` принимает итерируемый объект (список, кортеж) и записывает строки одну за другой. Важно, что он не добавляет автоматически `\n`, поэтому символы переноса нужно включать в сами строки.

### 1.5. Построчное чтение без загрузки всего файла в память

```python
FILE_PATH = "file.txt"

def main():
    with open(FILE_PATH, 'r') as f:
        # Итерируемся по файлу – это читает строки по одной, не загружая весь файл
        for line in f:
            # print по умолчанию добавляет свой перенос строки, а в line уже есть перенос
            # Чтобы не было двойного переноса, указываем end='' (пустой символ в конце)
            print(line, end='')
    # После цикла добавим ещё один пустой print для красоты
    print()

if __name__ == "__main__":
    main()
```

**Вывод в консоли:**
```
hello world
hello again
cats
dogs

```

**Пояснение:**
- Цикл `for line in f:` читает файл построчно, но в отличие от `readlines()` не хранит все строки в памяти одновременно. Это важно для больших файлов (логи, большие данные).
- `print(line, end='')` – убирает автоматический перенос строки, который `print` добавляет по умолчанию, потому что в `line` уже есть `\n`.
- После цикла печатаем пустую строку, чтобы отделить вывод.

## 2. Формат JSON

JSON (JavaScript Object Notation) – текстовый формат обмена данными, похожий на словари и списки Python. В Python есть встроенный модуль `json`.

### 2.1. Преобразование Python-объекта в JSON-строку (json.dumps)

```python
import json

def demo_json_dumps():
    data = {
        "users": [
            {"id": 42, "username": "John"},
            {"id": 43, "username": "Sam"}
        ]
    }
    print("Исходный объект:", data)
    print("Тип:", type(data))
    # Преобразуем в JSON-строку
    data_str = json.dumps(data)
    print("JSON-строка:", data_str)
    print("Тип JSON-строки:", type(data_str))

if __name__ == "__main__":
    demo_json_dumps()
```

**Вывод в консоли:**
```
Исходный объект: {'users': [{'id': 42, 'username': 'John'}, {'id': 43, 'username': 'Sam'}]}
Тип: <class 'dict'>
JSON-строка: {"users": [{"id": 42, "username": "John"}, {"id": 43, "username": "Sam"}]}
Тип JSON-строки: <class 'str'>
```

**Пояснение:**
- `json.dumps()` – сериализует Python-объект в строку в формате JSON. Ключи автоматически оборачиваются в двойные кавычки, числа остаются числами, строки – в двойных кавычках.
- Обратите внимание, что в JSON нет одинарных кавычек – только двойные.
- Для удобочитаемости можно добавить отступы: `json.dumps(data, indent=2)` – тогда вывод будет с переносами и пробелами.

### 2.2. Преобразование JSON-строки в Python-объект (json.loads)

```python
import json

def demo_json_loads():
    json_str = '{"users": [{"id": 42, "username": "John"}, {"id": 43, "username": "Sam"}]}'
    print("JSON-строка:", json_str)
    print("Тип:", type(json_str))
    data = json.loads(json_str)
    print("Python-объект:", data)
    print("Тип:", type(data))
    print("Первый пользователь:", data["users"][0])

if __name__ == "__main__":
    demo_json_loads()
```

**Вывод:**
```
JSON-строка: {"users": [{"id": 42, "username": "John"}, {"id": 43, "username": "Sam"}]}
Тип: <class 'str'>
Python-объект: {'users': [{'id': 42, 'username': 'John'}, {'id': 43, 'username': 'Sam'}]}
Тип: <class 'dict'>
Первый пользователь: {'id': 42, 'username': 'John'}
```

**Пояснение:**
- `json.loads()` – десериализует JSON-строку в Python-объект (словари, списки, строки, числа, булевы значения, None).
- Это удобно для получения данных из сети или из файла.

### 2.3. Запись JSON в файл (json.dump)

```python
import json

FILE_JSON = "file.json"

def demo_json_dump():
    data = {
        "users": [
            {"id": 42, "username": "John"},
            {"id": 43, "username": "Sam"}
        ]
    }
    # Записываем в файл с отступами для удобства чтения
    with open(FILE_JSON, 'w') as f:
        json.dump(data, f, indent=2)

if __name__ == "__main__":
    demo_json_dump()
```

**Результат:**
Создаётся файл `file.json` с содержимым:
```json
{
  "users": [
    {
      "id": 42,
      "username": "John"
    },
    {
      "id": 43,
      "username": "Sam"
    }
  ]
}
```

**Пояснение:**
- `json.dump(obj, file)` – записывает JSON-представление объекта прямо в файловый объект.
- Параметр `indent=2` добавляет форматирование (отступы), что удобно для чтения человеком.
- Этот способ эффективнее, чем сначала делать `dumps`, а потом `write`, потому что `dump` может записывать данные частями (для больших объектов).

### 2.4. Чтение JSON из файла (json.load)

```python
import json

FILE_JSON = "file.json"

def demo_json_load():
    with open(FILE_JSON, 'r') as f:
        data = json.load(f)
    print("Прочитанные данные:", data)
    print("Тип:", type(data))
    print("Первый пользователь:", data["users"][0])

if __name__ == "__main__":
    demo_json_load()
```

**Вывод:**
```
Прочитанные данные: {'users': [{'id': 42, 'username': 'John'}, {'id': 43, 'username': 'Sam'}]}
Тип: <class 'dict'>
Первый пользователь: {'id': 42, 'username': 'John'}
```

**Пояснение:**
- `json.load(file)` – читает весь файл и парсит его как JSON, возвращая Python-объект.
- Важно: если JSON большой, он целиком загружается в память. Для очень больших JSON-файлов нужно использовать потоковую обработку (например, `ijson`), но в стандартной библиотеке такого нет.

### 2.5. Формат JSON Lines (каждая строка – отдельный JSON)

Иногда удобно хранить в файле множество JSON-объектов, каждый на своей строке. Такой файл не является валидным JSON целиком, но каждая строка валидна.

```python
import json

FILE_JL = "file.jl"

def demo_json_lines():
    users = [
        {"id": 1, "name": "John", "friends": [2, 3]},
        {"id": 2, "name": "Sam", "friends": [1]},
        {"id": 3, "name": "Nick", "friends": [1]}
    ]
    # Запись: открываем в режиме добавления (append) или записи ('w')
    with open(FILE_JL, 'w') as f:
        for user in users:
            # Записываем JSON-строку и обязательно добавляем перенос строки
            json.dump(user, f)
            f.write('\n')

    # Чтение построчно
    with open(FILE_JL, 'r') as f:
        for line in f:
            line = line.strip()  # убираем символ переноса
            if line:  # если строка не пустая
                user_data = json.loads(line)
                print(f"ID: {user_data['id']}, Name: {user_data['name']}")

if __name__ == "__main__":
    demo_json_lines()
```

**Вывод в консоли:**
```
ID: 1, Name: John
ID: 2, Name: Sam
ID: 3, Name: Nick
```

**Пояснение:**
- При записи каждый JSON-объект записывается в отдельную строку. Обязательно добавляем `\n`.
- При чтении мы итерируемся по строкам файла, каждую строку десериализуем отдельно. Это позволяет обрабатывать огромное количество объектов без загрузки всего файла в память.
- Проверка `if line:` нужна, чтобы пропустить возможные пустые строки.

## 3. Формат CSV (Comma-Separated Values)

CSV – простой табличный формат, где строки разделены символами новой строки, а поля внутри строки – запятыми (или другими разделителями). Python имеет модуль `csv`.

### 3.1. Чтение CSV как списка строк

```python
import csv

FILE_CSV = "cars.csv"   # предположим, файл существует с данными

def read_csv_as_list():
    with open(FILE_CSV, 'r', newline='') as f:
        # создаём объект reader
        csv_reader = csv.reader(f)
        for row in csv_reader:
            print(row)

if __name__ == "__main__":
    read_csv_as_list()
```

**Пояснение:**
- Параметр `newline=''` в `open` рекомендуется для корректной обработки переносов строк в CSV.
- `csv.reader` возвращает итератор, который выдаёт строки как списки строк (поля не преобразуются в числа автоматически).
- Если первая строка – заголовок, она тоже попадёт в вывод.

**Пример вывода (если файл `cars.csv` с заголовками):**
```
['year', 'make', 'model', 'description', 'price']
['1997', 'Ford', 'E350', 'ac, abs, moon', '3000.00']
['1999', 'Chevy', 'Venture "Extended Edition"', '', '4900.00']
```

### 3.2. Чтение CSV как словарей (DictReader)

```python
import csv

FILE_CSV = "cars.csv"

def read_csv_as_dict():
    with open(FILE_CSV, 'r', newline='') as f:
        csv_reader = csv.DictReader(f)
        # Получить имена заголовков (fieldnames)
        print("Заголовки:", csv_reader.fieldnames)
        for row in csv_reader:
            # row – это словарь, ключи – заголовки
            print(row['make'], row['price'])

if __name__ == "__main__":
    read_csv_as_dict()
```

**Пояснение:**
- `csv.DictReader` автоматически использует первую строку файла как ключи (заголовки). Если заголовков нет, их можно передать параметром `fieldnames`.
- Каждая строка возвращается в виде словаря, что удобно для доступа по имени колонки.
- Значения всегда строки; числовые преобразования нужно делать вручную.

**Пример вывода:**
```
Заголовки: ['year', 'make', 'model', 'description', 'price']
Ford 3000.00
Chevy 4900.00
```

### 3.3. Запись CSV с помощью DictWriter

```python
import csv

FILE_CSV_OUT = "birthmonths.csv"

def write_csv_dict():
    fieldnames = ['name', 'month', 'department']
    data = [
        {'name': 'Sam', 'month': 'January', 'department': 'IT'},
        {'name': 'Jack', 'month': 'May', 'department': 'Help Desk'},
        {'name': 'Kate', 'month': 'March', 'department': 'Accounting'}
    ]

    with open(FILE_CSV_OUT, 'w', newline='') as f:
        writer = csv.DictWriter(f, fieldnames=fieldnames)
        # Записываем заголовок
        writer.writeheader()
        # Записываем строки по одной
        for row in data:
            writer.writerow(row)
        # Или можно сразу несколько: writer.writerows(data)

if __name__ == "__main__":
    write_csv_dict()
```

**Результат:**
Создаётся файл `birthmonths.csv` с содержимым:
```
name,month,department
Sam,January,IT
Jack,May,Help Desk
Kate,March,Accounting
```

**Пояснение:**
- `csv.DictWriter` принимает файловый объект и список имён полей (`fieldnames`).
- `writeheader()` записывает строку с заголовками.
- `writerow(dict)` записывает одну строку, `writerows(list_of_dicts)` – несколько.
- Важно: порядок полей в словаре может быть любым, `DictWriter` использует `fieldnames` для определения порядка колонок в файле.

## 4. Формат INI (конфигурационные файлы)

Файлы `.ini` содержат секции `[Section]` и пары ключ=значение. Для работы с ними используется модуль `configparser`.

### 4.1. Чтение INI-файла

```python
import configparser

FILE_INI = "demo_config.ini"

def read_ini():
    config = configparser.ConfigParser()
    # Читаем файл
    config.read(FILE_INI)
    # Выводим все секции (по умолчанию секция DEFAULT не отображается в sections())
    print("Секции:", config.sections())
    # Получаем значение из секции DEFAULT
    environ = config['DEFAULT'].get('environ')
    print("environ =", environ)
    # Получаем значение из секции mysql, преобразуя к int
    port = config['mysql'].getint('port')
    print("port =", port, type(port))
    # Получаем значение из секции mysql как строку
    host = config['mysql'].get('host')
    print("host =", host)

if __name__ == "__main__":
    read_ini()
```

**Предположим, файл `demo_config.ini` выглядит так:**
```ini
[DEFAULT]
environ = development
secret_key = 12345

[mysql]
host = localhost
port = 3306
user = root

[files]
home = /home/user
```

**Вывод в консоли:**
```
Секции: ['mysql', 'files']
environ = development
port = 3306 <class 'int'>
host = localhost
```

**Пояснение:**
- `configparser.ConfigParser()` создаёт объект для работы с INI.
- `config.read(filename)` читает файл (можно передать несколько файлов).
- Секция `DEFAULT` всегда существует, но не включается в `config.sections()`. Доступ к её значениям можно получить через `config['DEFAULT']`.
- Для получения значений с преобразованием типов есть методы `getint()`, `getfloat()`, `getboolean()`.
- Значения из `DEFAULT` наследуются другими секциями, если в секции не переопределены.

### 4.2. Изменение и запись INI-файла

```python
import configparser

FILE_INI = "demo_config.ini"

def modify_ini():
    config = configparser.ConfigParser()
    config.read(FILE_INI)

    # Изменяем порт в секции mysql (преобразуем обратно в строку)
    config['mysql']['port'] = str(config['mysql'].getint('port') + 1)
    # Добавляем новое поле в секцию files
    config['files']['home'] = '/home/suren'

    # Записываем обратно в файл
    with open(FILE_INI, 'w') as f:
        config.write(f)

if __name__ == "__main__":
    modify_ini()
```

**Пояснение:**
- Объект `config` ведёт себя как словарь: `config['section']['key'] = value`. Все значения должны быть строками.
- Метод `write(file)` записывает конфигурацию в файл, форматируя его стандартным образом (с отступами, в алфавитном порядке ключей, если не переопределено).
- После выполнения файл `demo_config.ini` будет обновлён: порт увеличен на 1, добавлена запись `home = /home/suren`.

При использовании режима `'w'` файл **полностью перезаписывается**. Однако в данном коде объект `config` сначала считывает исходный файл, затем вы вносите изменения в этот объект, а потом записываете **весь объект обратно** в файл. В результате в файл попадают **все данные, которые были в объекте**: и исходные, и изменённые. Никакие данные из исходного файла не теряются (кроме тех, которые вы явно удалили или перезаписали в объекте).

**Ключевые моменты:**
- `config.read(FILE_INI)` загружает всё содержимое файла в память.
- После изменений `config['mysql']['port'] = ...` и `config['files']['home'] = ...` объект `config` содержит **все** секции и ключи, которые были в файле, плюс изменения.
- `open(FILE_INI, 'w')` открывает файл для записи, и при записи **стирает предыдущее содержимое**, но затем `config.write(f)` записывает **текущее состояние** объекта, то есть полную актуальную конфигурацию.

Таким образом, старый файл заменяется новым, который включает в себя всё, что было в исходном, но с учётом ваших изменений. Никакие данные не теряются, если только вы не удалили секцию или ключ в объекте перед записью.

**Если бы вы открыли файл в режиме добавления `'a'`**, то новая запись добавилась бы в конец, но это не имело бы смысла, так как `config.write()` ожидает, что файл пуст или уже содержит корректную INI-структуру для перезаписи.
