### Особенности для разных ОС

#### Linux (Ubuntu)
Автор:
**«Для начала, если вы хотите установить Python на Linux, например Ubuntu, и у вас еще не установлен Python, хотя обычно он идет в комплекте с всеми современными версиями Ubuntu, то вам нужно будет сделать команду sudo apt install python3. Но перед этим обязательно убедитесь, что у вас Python не установлен. Чтобы это проверить, достаточно выполнить команду python --version.»**

*Вывод в консоли (автор не показывает конкретно, но подразумевает):*
```bash
$ python --version
Python 3.10.4
```

#### Windows (рекомендация WSL)
Автор:
**«Если вы используете Windows, что с большой долей вероятности именно так, то вам необходимо установить систему WSL. WSL -- это система Windows Subsystem for Linux, которая позволяет запускать Linux, например, Ubuntu прямо на Windows. На сайте Microsoft есть также инструкция на русском, поэтому вы можете ее беспрепятственно пройти и настроить у себя на Windows систему для запуска Linux. Зачем это нужно? Да, Python можно установить прямо на Windows, но есть некоторые пакеты, с которыми мы будем дальше работать, которые могут не подходить под Windows, либо там нужно использовать альтернативы, либо там просто нет решения для конкретной задачи. Поэтому лучше всего использовать именно Linux. Также это поможет в дальнейшем при деплое. То есть, так как все приложения в основном будут в итоге запускаться на Linux, то лучше всего изначально и разрабатывать на Linux.
Если вы установите себе WSL, то вы сможете запускать Bash и сам Ubuntu прямо в Windows в отдельном окне.»**

Для старых Windows (7) предлагается VirtualBox с Ubuntu (Kubuntu LTS).

#### Mac
Автор:
**«Если вы работаете на Mac, например, это обычный Mac современных версий, то на него можно установить систему Homebrew. Тема Homebrew – это такой пакетный менеджер, который позволяет устанавливать различные пакеты через терминал, точно так же, как мы это делаем на Linux. Чтобы установить Python 3 версии, нужно после установки Homebrew еще написать "brew install python" и установится Python. Чтобы проверить, какая версия Python установлена, нужно точно так же написать в терминале "python --version" и будет выведена версия Python.»**

*Вывод:*
```bash
$ python --version
Python 3.10.4
```

## 5. Виртуальное окружение

### Что это и зачем
Автор:
**«Виртуальное окружение может быть активировано по-разному. ... Зачем нужно виртуальное окружение? При разработке на Python мы будем устанавливать различные пакеты извне, то есть мы будем устанавливать их из интернета. В зависимости от того, какая версия пакет установлена, в этом пакете есть различные функции. Мы можем работать не над одним проектом, мы будем работать над множеством проектов всегда. Будь то фриланс или какие-то личные проекты или работа, у нас будет всегда несколько проектов. Чтобы зависимости между проектами не пересекались, нужно использовать виртуальное окружение. Из названия понятно, что это является окружением, то есть какой-то средой, где у нас выполняются программы, и виртуальным оно является, потому что оно не настоящее, а просто условное. Почему оно условное? Потому что это на самом деле никакое не окружение, а папка.»**

### Создание виртуального окружения через терминал
Автор показывает, как создать виртуальное окружение с именем `myenv`:

*Проверка, что `venv` доступен:*
```bash
$ python3 -m venv --help
usage: venv [-h] [--system-site-packages] [--symlinks | --copies] [--clear] ...
```

*Создание:*
```bash
$ python3 -m venv myenv
```

*После создания появляется папка `myenv`:*
```bash
$ ls
myenv
```

*Просмотр содержимого `myenv/bin` (Linux/Mac):*
```bash
$ ls myenv/bin
activate   activate.csh   activate.fish   pip   pip3   python   python3
```

*Для Windows будет папка `Scripts` с файлами `activate.bat`, `Activate.ps1` и т.д.

### Активация виртуального окружения
Автор:
**«Теперь мы можем активировать это виртуальное окружение. ... Нам нужно открыть терминал, в данном случае мы уже в терминале, и затем воспользоваться скриптом, который есть в папке myenv /bin.»**

*Активация:*
```bash
$ source myenv/bin/activate
(myenv) $
```
В терминале появляется префикс `(myenv)`, показывающий, что окружение активировано.

### Проверка, какой Python используется
Автор:
**«Если при активированном виртуальном окружении спросить, какой питон используется командой which python, то мы увидим, что мы используем myenv bin python.»**

*Вывод:*
```bash
(myenv) $ which python
/Users/suren/pycharm-projects/python-project/myenv/bin/python
```

### Деактивация
Автор:
**«Чтобы деактивировать виртуальное окружение, нужно написать команду deactivate и виртуальное окружение деактивировано.»**

*Вывод (префикс исчезает):*
```bash
(myenv) $ deactivate
$
```

### Удаление виртуального окружения
Автор:
**«Как выполняется удаление в терминале? Для этого есть команда rm, которая удаляет безвозвратно, без корзины.»**

*Удаление файла:*
```bash
$ rm myenv/pyvenv.cfg
```

*Удаление всей папки рекурсивно:*
```bash
$ rm -r myenv
```

### Создание виртуального окружения через PyCharm
Автор описывает, как в PyCharm добавить интерпретатор:

Мы выбираем через какой инструмент это делаем, и мы делаем это через virtual environment, то есть через стандартный инструмент, и выбираем, где создать виртуальное окружение. Мы его создаем здесь в slash venv.»**
