## Шаг 0. Главная мысль (запомни её)

> **В Python всё — объект.**
> Объект создаётся от **класса**.
> А сам класс — это тоже объект, он создаётся от **метакласса**.
> Метакласс по умолчанию — `type`.

Визуально:
`экземпляр (instance) → класс (class) → метакласс (metaclass)`

---

## Шаг 1. Обычный класс и проверка его «типа»

Создадим файл `demo_class.py`.

```python
class Foo:
    """My Foo class"""
    foobar = "spam and eggs"

    @classmethod
    def echo_name(cls):
        print(cls.__name__)
```

Что здесь происходит:

- `class Foo:` — объявляем класс. Python **создаёт объект класса** в памяти.
- `foobar` — атрибут класса (общий для всех экземпляров).
- `@classmethod` — декоратор, превращающий метод в метод класса. Вместо `self` он получает `cls` — сам класс.

Теперь добавим выводы:

```python
print(Foo)                     # 1. Что такое Foo?
print(Foo.mro())               # 2. Цепочка наследования
foo = Foo()                    # 3. Создаём экземпляр
print(foo)                     # 4. Что такое foo?
foo.echo_name()                # 5. Вызов метода класса через экземпляр

print("type foo:", type(foo))  # 6. Тип экземпляра
print("type Foo:", type(Foo))  # 7. Тип класса
print("type of type:", type(type))  # 8. Тип самого type
```

**Вывод в консоли:**

```
<class '__main__.Foo'>
[<class '__main__.Foo'>, <class 'object'>]
<__main__.Foo object at 0x10391cd70>
Foo
type foo: <class '__main__.Foo'>
type Foo: <class 'type'>
type of type: <class 'type'>
```

**Анализ по шагам:**

1. `Foo` — это класс, принадлежащий модулю `__main__`.
2. `Foo.mro()` показывает: `Foo` → `object` (все классы в Python 3 наследуются от `object` неявно).
3. `foo` — экземпляр класса `Foo`.
4. При вызове `foo.echo_name()` выводится `Foo` — имя класса.
5. `type(foo)` — класс `Foo`.
6. **Ключевой момент:** `type(Foo)` — это **`type`**. То есть класс `Foo` является экземпляром метакласса `type`.
7. `type(type)` — тоже `type`. Это вершина.

**Вывод:** любой обычный класс создаётся метаклассом `type`.

## Шаг 2. Создание класса «на лету» через `type()` (динамически)

Можно создать класс, не используя ключевое слово `class`, а вызвав `type` как конструктор.

Синтаксис:
`type(имя_класса, кортеж_родителей, словарь_атрибутов)`

Создадим файл `demo_create_type.py`.

```python
# Определим функции, которые станут методами класса
def echo_name(cls):
    print(cls.__name__)

def echo_foobar(self):
    print("foobar value:", self.foobar)

# Имя атрибута будем задавать через переменную
attr_name = "foobar"

# Динамически создаём класс
Foo = type("Foo", (), {
    attr_name: "spam and eggs",          # атрибут foobar
    "echo_name": echo_name,              # метод
    "echo_foobar": echo_foobar,          # метод
})
```

Обрати внимание:
- В словаре ключи — это строки, которые станут именами атрибутов класса.
- Значения могут быть любыми объектами: строками, функциями, числами и т.д.

Теперь проверим:

```python
print(Foo)                # что такое Foo?
print(Foo.mro())          # цепочка наследования
print(type(Foo))          # тип Foo

foo = Foo()               # создаём экземпляр
print(foo)                # что такое foo?
Foo.echo_name()           # вызываем метод класса
foo.echo_foobar()         # вызываем метод экземпляра

print(foo.__dict__)       # атрибуты экземпляра (пусто)
print(dir(Foo))           # список всех атрибутов класса
print(Foo.foobar)         # значение атрибута класса
```

**Вывод в консоли:**

```
<class '__main__.Foo'>
[<class '__main__.Foo'>, <class 'object'>]
<class 'type'>
<__main__.Foo object at 0x1028b4d70>
Foo
foobar value: spam and eggs
{}
['__class__', '__delattr__', ..., 'echo_foobar', 'echo_name', 'foobar']
spam and eggs
```

**Что важно:**

- `type(Foo)` снова `type` — метакласс не изменился.
- Класс `Foo`, созданный через `type`, ничем не отличается от класса, созданного через `class`.
- Мы смогли подставить имя атрибута `foobar` из переменной — это и есть **динамическое создание**.

**Зачем это нужно?**
Например, вы пишете код, который генерирует классы на основе данных из базы или конфигурации. Вы заранее не знаете, какие атрибуты и методы будут нужны — вы их собираете в словарь и вызываете `type`.

## Шаг 3. Создаём **свой метакласс** (наследуем от `type`)

Сделаем файл `demo_my_metaclass.py`.
Метакласс — это класс, который наследуется от `type`. Он может переопределять специальные методы, например `__new__`.

### Что такое `__new__` в метаклассе?

- `__new__` вызывается **до** создания объекта (в данном случае — до создания класса).
- Он получает:
  `cls` — сам метакласс,
  `name` — имя нового класса,
  `bases` — кортеж родительских классов,
  `dct` — словарь атрибутов (то, что будет внутри класса).
- Он **обязан** вернуть новый объект класса (обычно через `super().__new__`).

Простейший метакласс:

```python
class MyMetaclass(type):
    def __new__(cls, name, bases, dct, *args, **kwargs):
        print("Создаём класс", name)
        print("Родители:", bases)
        print("Атрибуты:", dct)
        # Создаём класс стандартным способом
        new_class = super().__new__(cls, name, bases, dct, *args, **kwargs)
        return new_class
```

Обрати внимание: `*args, **kwargs` нужны на случай, если в будущем метаклассу передадут дополнительные параметры.

## Шаг 4. Используем свой метакласс — первый способ (явный вызов)

Можно вызвать метакласс как конструктор, аналогично `type`:

```python
Foo = MyMetaclass("Foo", (), {"spam": "eggs"})
```

Выполним этот код (в том же файле после определения метакласса).

## Шаг 5. Добавляем логику в метакласс — модифицируем атрибуты

Автор в видео делает следующее:
- Проходит по всем ключам в `dct` (словаре атрибутов).
- Если ключ начинается с одного подчёркивания `_` **и** не заканчивается на `__` (чтобы не трогать магические методы типа `__init__`), то он удаляет старый ключ и добавляет новый — с именем в верхнем регистре, а значение оставляет тем же.
- Также добавляет новый атрибут `SPAM_AND_EGGS = "foobar"` в каждый создаваемый класс.

Вот код метакласса с этой логикой:

```python
class MyMetaclass(type):
    def __new__(cls, name, bases, dct, *args, **kwargs):
        print("new class", name)
        print("with bases:", bases)
        print("and attrs", dct)

        # Модифицируем словарь атрибутов
        # Идём по списку ключей (list(dct.keys())), чтобы менять словарь во время итерации
        for key in list(dct.keys()):
            if key.startswith("_") and not key.endswith("__"):
                value = dct.pop(key)           # удаляем старый ключ
                dct[key.upper()] = value       # добавляем новый с именем в верхнем регистре

        # Добавляем свой атрибут
        dct["SPAM_AND_EGGS"] = "foobar"

        print("dct after modification:", dct)

        # Создаём класс с изменённым словарём
        new_class = super().__new__(cls, name, bases, dct, *args, **kwargs)
        return new_class
```

Теперь создадим класс через этот метакласс:

```python
Foo = MyMetaclass("Foo", (), {"spam": "eggs", "_secret": "hidden"})
```

**Вывод в консоли:**

```
new class Foo
with bases: ()
and attrs {'spam': 'eggs', '_secret': 'hidden'}
dct after modification: {'spam': 'eggs', 'SECRET': 'hidden', 'SPAM_AND_EGGS': 'foobar'}
```

Обрати внимание:
- `_secret` превратился в `SECRET`.
- Добавился `SPAM_AND_EGGS`.

Проверим:

```python
print(Foo.spam)        # eggs
print(Foo.SECRET)      # hidden
print(Foo.SPAM_AND_EGGS)  # foobar
```

Всё работает.

## Шаг 6. Второй способ использования метакласса — параметр `metaclass=` при объявлении класса

Вместо вызова `MyMetaclass(...)` мы можем написать обычный `class`, но указать `metaclass=MyMetaclass`.

```python
class Foo(metaclass=MyMetaclass):
    spam = "eggs"
    _secret = "hidden"
```

Python автоматически создаст класс `Foo`, используя наш метакласс.

**Вывод (аналогичен предыдущему):**

```
new class Foo
with bases: ()
and attrs {'__module__': '__main__', '__qualname__': 'Foo', 'spam': 'eggs', '_secret': 'hidden', ...}
dct after modification: {'__module__': '__main__', '__qualname__': 'Foo', 'spam': 'eggs', 'SECRET': 'hidden', 'SPAM_AND_EGGS': 'foobar', ...}
```

Теперь `type(Foo)` снова будет `MyMetaclass`.

**Важно:** Если в классе есть методы (например, `__init__`), они тоже попадают в `dct`. Но нужно быть осторожным: изменение имён методов может сломать код. Автор видео намеренно отсекает ключи, заканчивающиеся на `__`, чтобы не трогать магические методы.

---

# Полный код и важный момент

```python
class MyMetaclass(type):
    def __new__(cls, name, bases, dct, *args, **kwargs):
        print("new class", name)
        print("with bases:", bases)
        print("and attrs", dct)

        print("dct before:", dct)

        for key in list(dct.keys()):
            # if key.startswith("__") and not key.endswith("__"):
            if key.startswith("_") and not key.endswith("__"):
                value = dct.pop(key)
                # dct[key[2:].upper()] = value
                dct[key.upper()] = value

        print("dct:", dct)
        dct["SPAM_AND_EGGS"] = "foobar"
        new_class = super().__new__(cls, name, bases, dct, *args, **kwargs)
        return new_class

# Foo = MyMetaclass("Foo", (), {})
Foo = MyMetaclass("Foo", (), {"spam": "eggs"})

print(Foo)
print(Foo.mro())
print(MyMetaclass.mro(Foo))
print(type(Foo))
print(dir(Foo))
print(Foo.spam)

print("Foo.SPAM_AND_EGGS", Foo.SPAM_AND_EGGS)


class Foo(object, metaclass=MyMetaclass):
    def __new__(cls, *args, **kwargs):
        print("__new__class", cls)

    def __init__(self):
        self.__super_secret_value = "fizzbuzz"

    # __super_secret_value = "fizzbuzz"
    _another_secret_value = "fizzbuzz"


print(Foo)
print(type(Foo))
print(dir(Foo))

print("Foo.SPAM_AND_EGGS", Foo.SPAM_AND_EGGS)

```

## Главная идея (запомни)

**Метакласс работает ТОЛЬКО с атрибутами класса.**
Атрибуты, которые создаются внутри методов (например, внутри `__init__` через `self.что-то = значение`), метакласс **не видит** и **не может изменить**.

Почему? Потому что метакласс срабатывает в момент **создания класса**, когда методов (в том числе `__init__`) ещё нет как исполняемого кода. Он видит только то, что объявлено на уровне класса.

---

## Шаг 1. Атрибут класса vs атрибут экземпляра

Рассмотрим простой пример без метаклассов.

```python
class Example:
    class_attr = "я атрибут класса"          # объявлен на уровне класса

    def __init__(self):
        self.instance_attr = "я атрибут экземпляра"  # создаётся при создании объекта
```

- `class_attr` существует в словаре класса (`Example.__dict__`). Он один для всех экземпляров.
- `instance_attr` не существует в классе. Он появляется только после вызова `__init__` у конкретного экземпляра и хранится в `obj.__dict__`.

Проверим:

```python
print(Example.__dict__.keys())   # ... 'class_attr' ...
obj = Example()
print(obj.__dict__)              # {'instance_attr': 'я атрибут экземпляра'}
```

**Вывод:**
`dict_keys(['__module__', 'class_attr', ...])`
`{'instance_attr': 'я атрибут экземпляра'}`

Видишь разницу? `class_attr` попал в словарь класса, а `instance_attr` — только в словарь экземпляра.

## Шаг 2. Что видит метакласс в `dct`?

Когда мы создаём класс (через `class ...` или через `type`), Python собирает все **атрибуты класса** в словарь и передаёт его в метакласс как параметр `dct`.

В этот словарь **не попадают**:
- Атрибуты, которые создаются внутри методов (потому что методы ещё не выполнялись).
- Локальные переменные внутри методов.

**Поэтому** в коде из видео:

```python
class Foo(object, metaclass=MyMetaclass):
    def __new__(cls, *args, **kwargs):
        print("__new__class", cls)

    def __init__(self):
        self.__super_secret_value = "fizzbuzz"

    _another_secret_value = "fizzbuzz"
```

Что попадёт в `dct`?

- `__new__` (метод) — попадёт, потому что это атрибут класса (функция, объявленная на уровне класса).
- `__init__` (метод) — тоже попадёт.
- `_another_secret_value = "fizzbuzz"` — **попадёт**, это атрибут класса.
- А вот `self.__super_secret_value = "fizzbuzz"` внутри `__init__` — **НЕ попадёт**, потому что это код, который выполнится только при создании экземпляра, а не при создании класса.

## Шаг 3. Почему `__super_secret_value` не изменился?

Твой вопрос был: почему в выводе мы не видим изменения `__super_secret_value` (превращения в `__SUPER_SECRET_VALUE`)?

Ответ: **потому что его нет в `dct`.**
Метакласс проходит по ключам `dct` и модифицирует только те, которые там есть. А `__super_secret_value` как атрибут класса не объявлен — он закомментирован в коде автора:

```python
# __super_secret_value = "fizzbuzz"   # <- закомментировано!
```

То есть строчка, которая могла бы создать атрибут класса `__super_secret_value`, не активна. Вместо этого `__super_secret_value` появляется только внутри `__init__` как атрибут экземпляра.

### Давай проверим экспериментально

Добавим в класс явный атрибут класса `__super_secret_value` (раскомментируем) и посмотрим, что случится.

**Исходный код (с закомментированной строкой):**

```python
class Foo(metaclass=MyMetaclass):
    def __init__(self):
        self.__super_secret_value = "fizzbuzz"   # атрибут экземпляра
    # __super_secret_value = "fizzbuzz"         # закомментирован - атрибут класса
```

**Что видит метакласс в `dct`?**
Вывод из консоли (приведён в исходном сообщении, фрагмент):

```
and attrs {'__module__': '__main__', '__qualname__': 'Foo', ..., '_another_secret_value': 'fizzbuzz', ...}
```

Там нет `__super_secret_value`. Поэтому метакласс его не обрабатывает.

**А теперь раскомментируем:**

```python
class Foo(metaclass=MyMetaclass):
    def __init__(self):
        self.__super_secret_value = "fizzbuzz"
    __super_secret_value = "fizzbuzz"   # теперь это атрибут класса!
```

Тогда в `dct` появится ключ `__super_secret_value`. И метакласс его увидит. Применит условие:
`key.startswith("_")` — да, начинается с `_` (первый символ — подчёркивание).
`not key.endswith("__")` — `__super_secret_value` заканчивается на `e`, а не на `__`, значит условие истинно.
Метакласс удалит `__super_secret_value` и добавит `__SUPER_SECRET_VALUE` (преобразуя в верхний регистр). Но тут важно: имя станет `__SUPER_SECRET_VALUE` (два подчёркивания в начале останутся, потому что мы делаем `key.upper()`, а не `key[2:].upper()`). В оригинале автор закомментировал вариант с `key[2:].upper()` и использует просто `key.upper()`. Так что получится `__SUPER_SECRET_VALUE`.

---

## Шаг 4. Почему в выводе всё же есть `_ANOTHER_SECRET_VALUE`?

А вот `_another_secret_value` — это атрибут класса (объявлен на уровне класса). Он попал в `dct`, и метакласс его преобразовал в `_ANOTHER_SECRET_VALUE` (одно подчёркивание + большие буквы). Поэтому мы можем написать:

```python
print(Foo._ANOTHER_SECRET_VALUE)   # выведет 'fizzbuzz'
```

А `__super_secret_value` как атрибут экземпляра существует только у объекта `foo`, и его имя искажается (name mangling) из-за двух подчёркиваний в начале: оно превращается в `_Foo__super_secret_value`. Но это уже совсем другая история (приватные атрибуты), и метакласс к этому не имеет отношения.

---

## Шаг 5. Вывод из консоли — подтверждение

Вот фрагмент вывода из твоего сообщения (я отформатировал для ясности):

```
new class Foo
with bases: (<class 'object'>,)
and attrs {'__module__': '__main__', '__qualname__': 'Foo', '__firstlineno__': 34,
           '__new__': <function Foo.__new__ at 0x...>,
           '__init__': <function Foo.__init__ at 0x...>,
           '_another_secret_value': 'fizzbuzz',   # <-- атрибут класса
           '__static_attributes__': ('__super_secret_value',),
           '__classdictcell__': <cell ...>}
dct before: { ... }
dct: { ... '_ANOTHER_SECRET_VALUE': 'fizzbuzz' ...}   # изменился
```

Видишь: в `dct` был `_another_secret_value`, а стал `_ANOTHER_SECRET_VALUE`.
А `__super_secret_value` **вообще отсутствует** в `dct`, потому что он не был объявлен как атрибут класса.

---

## Итог (самое важное)

1. **Метакласс работает только со словарём атрибутов класса (`dct`)**.
2. **Атрибуты экземпляра** (те, что создаются через `self.attr = value` в методах) **не попадают в `dct`** и не могут быть изменены метаклассом.
3. Чтобы метакласс "увидел" атрибут, он должен быть объявлен на уровне класса (прямо внутри `class`, но вне методов).
4. В коде автора `__super_secret_value` был атрибутом экземпляра, поэтому его не было в `dct`, и метакласс его не трогал.

## Теперь закрепим понимание — сделай сам маленький эксперимент

Напиши такой код:

```python
class MyMeta(type):
    def __new__(cls, name, bases, dct):
        print("Ключи в dct:", list(dct.keys()))
        return super().__new__(cls, name, bases, dct)

class Test(metaclass=MyMeta):
    class_attr = "я в dct"
    def __init__(self):
        self.instance_attr = "меня нет в dct"
```

Запусти и посмотри, что выведет метакласс. Ты увидишь `class_attr`, но не увидишь `instance_attr`.

Потом добавь внутри `Test` строчку:

```python
    __secret = "атрибут класса с двумя подчёркиваниями"
```

И снова запусти — теперь `__secret` появится в `dct`. Метакласс может его изменить, если ты добавишь логику.

## Шаг 7. Реальный пример — абстрактные классы (`abc.ABCMeta`)

В стандартной библиотеке `abc` есть метакласс `ABCMeta`. Он позволяет создавать абстрактные базовые классы, которые требуют от наследников реализовать определённые методы.

Создадим файл `demo_meta_abc.py`.

```python
from abc import ABC, abstractmethod

class FileManagerABC(ABC):
    @abstractmethod
    def read_file(self):
        pass
```

- `ABC` — это класс, у которого метаклассом является `ABCMeta`.
- `@abstractmethod` — декоратор, который помечает метод как абстрактный.
- Метакласс `ABCMeta` при создании класса проверяет: если в классе есть абстрактные методы, то создать экземпляр такого класса нельзя, пока их не реализовать в наследнике.

Проверим тип:

```python
print(type(FileManagerABC))  # <class 'abc.ABCMeta'>
```

Теперь создадим наследника, который **реализует** `read_file`:

```python
class FileManager(FileManagerABC):
    def __init__(self, filename):
        self.filename = filename

    def read_file(self):
        print("read file", self.filename)

if __name__ == "__main__":
    fm = FileManager("data.txt")
    print(fm)            # <__main__.FileManager object at ...>
    fm.read_file()       # read file data.txt
```

**Вывод:**

```
<class '__main__.FileManagerABC'>
<__main__.FileManager object at 0x...>
read file data.txt
```

Если бы мы **не реализовали** `read_file` в `FileManager`, то при попытке создать экземпляр получили бы ошибку:

```
TypeError: Can't instantiate abstract class FileManager with abstract method read_file
```

Именно метакласс `ABCMeta` генерирует эту ошибку при создании объекта (в своём `__new__` или `__init__` он проверяет, все ли абстрактные методы переопределены).

## Шаг 8. Когда же **реально** нужны свои метаклассы?

Автор прав в 99% случаев — они **не нужны**. Но есть сценарии:

1. **Регистрация классов** — автоматически добавлять каждый созданный класс в глобальный словарь.
2. **Проверка корректности** — например, убедиться, что все методы класса имеют docstring или аннотации типов.
3. **Автоматическое добавление методов** (как в Django ORM, где вы объявляете `class Meta`, а метакласс создаёт множество вспомогательных методов).
4. **Создание классов на основе данных** (динамически, как мы делали через `type`).

Но прежде чем писать свой метакласс, подумай: может быть, можно обойтись декоратором, наследованием или дескриптором? Метаклассы делают код неявным и сложным для понимания.
