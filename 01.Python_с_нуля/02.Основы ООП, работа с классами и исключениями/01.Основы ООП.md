## 1. Docstring (строки документации)

В Python **docstring** — это многострочная строка, которая идёт сразу после объявления модуля, функции, класса или метода. Она служит для документации и доступна через атрибут `__doc__`.

### Docstring модуля

Если написать многострочную строку в начале файла (без присваивания переменной), она станет docstring модуля.

```python
# файл step1.py
"""Это docstring для модуля step1.
Он объясняет, для чего нужен этот файл."""
```

PyCharm выделяет такие строки серым цветом (если настройки по умолчанию).

### Docstring функции

```python
def my_func(a, b):
    """This is my_func docstring.
    This function swaps a and b.
    Example: my_func(1, 2) -> (2, 1)
    """
    return b, a
```

**Как посмотреть docstring**:
- В интерактивной среде: `help(my_func)`
- В коде: `print(my_func.__doc__)`

```python
print(my_func.__doc__)
```

**Вывод** (с сохранением отступов и пустых строк):
```
This is my_func docstring.
    This function swaps a and b.
    Example: my_func(1, 2) -> (2, 1)

```

**Важно:** Если после заголовка функции сразу идёт не строка, а другой код (например, `a = 1`), то строка ниже уже не будет docstring'ом.

### Docstring как тело функции

В Python функция не может быть пустой — нужно что-то написать. Можно использовать `pass` или docstring (он тоже считается телом). Например:

```python
def hello():
    """This is hello docstring."""
    # тело функции пустое, но docstring есть

print(hello())  # None
print(hello.__doc__)  # "This is hello docstring."
```

### Docstring класса

Аналогично: сразу после `class ClassName:` пишем строку.

```python
class User:
    """Класс, представляющий пользователя системы."""
    pass

print(User.__doc__)  # "Класс, представляющий пользователя системы."
```

### Доступ к docstring через `__doc__`

```python
def foo():
    """Returns 42."""
    return 42

print(foo.__doc__)   # "Returns 42."
```

Если docstring нет, то `__doc__` будет `None`.

## 2. Основы классов: создание, атрибуты, экземпляры

В Python **всё является объектом**. Каждый объект имеет тип (класс). Класс — это "чертёж" для создания объектов (экземпляров).

### Простейший класс

```python
class User:
    pass

print(User)          # <class '__main__.User'>
```

Создадим экземпляр:

```python
user1 = User()
print(user1)         # <__main__.User object at 0x...>
print(type(user1))   # <class '__main__.User'>
```

Убедимся, что класс `User` наследуется от `object` (даже если не указали явно):

```python
print(User.__bases__)  # (<class 'object'>,)
# или
print(User.mro())      # [<class '__main__.User'>, <class 'object'>]
```

### Динамическое добавление атрибутов

Python позволяет добавлять атрибуты объекту «на лету»:

```python
user1.name = "John"
print(user1.name)      # John

user2 = User()
user2.name = "Sam"
user2.age = 20

print(user2.name, user2.age)  # Sam 20
```

Но если мы попытаемся обратиться к `user1.age`, получим ошибку:

```python
# print(user1.age)  # AttributeError: 'User' object has no attribute 'age'
```

### Словарь `__dict__`

Каждый объект хранит свои атрибуты в словаре `__dict__`:

```python
print(user1.__dict__)   # {'name': 'John'}
print(user2.__dict__)   # {'name': 'Sam', 'age': 20}
```

### Атрибуты класса

Если мы хотим, чтобы у всех экземпляров по умолчанию были какие-то атрибуты, можно определить их на уровне класса:

```python
class User:
    name = None   # атрибут класса
    age = None

u1 = User()
u2 = User()

print(u1.name)   # None (берётся из класса)
print(u2.name)   # None

u1.name = "John" # теперь у экземпляра появился свой атрибут name
print(u1.__dict__)  # {'name': 'John'}
print(u2.__dict__)  # {} (у u2 нет своего name, он всё ещё берёт из класса)
```

**Важно:** когда мы читаем `u1.name`, Python сначала ищет атрибут в экземпляре (`__dict__`), если не находит — идёт в класс, потом в родительские классы. При записи (`u1.name = ...`) атрибут создаётся именно в экземпляре, не затрагивая класс.

Если изменить атрибут класса через сам класс, то все экземпляры, у которых нет своего атрибута, увидят новое значение:

```python
User.age = 100
print(u1.age)   # 100 (т.к. у u1 нет своего age)
print(u2.age)   # 100
```

Но если у экземпляра уже есть свой атрибут, он не изменится:

```python
u2.age = 25
User.age = 200
print(u2.age)   # 25 (свой)
print(u1.age)   # 200 (берётся из класса)
```

## 3. Инициализатор `__init__`

Инициализатор вызывается автоматически при создании экземпляра. Он позволяет задать начальные атрибуты. Первый параметр — `self` (ссылка на сам создаваемый объект).

```python
class User:
    def __init__(self, name, age):
        self.name = name
        self.age = age

user = User("Alice", 30)
print(user.name, user.age)  # Alice 30
print(user.__dict__)        # {'name': 'Alice', 'age': 30}
```

Теперь у каждого пользователя **обязательно** будут атрибуты `name` и `age`, даже если мы их не задали явно после создания. Это предотвращает ошибки типа `AttributeError`.

### Параметры инициализатора

Параметры `__init__` могут быть любыми: позиционные, именованные, со значениями по умолчанию.

```python
class Product:
    def __init__(self, name, price=0):
        self.name = name
        self.price = price

laptop = Product("Laptop", 2000)
print(laptop.__dict__)  # {'name': 'Laptop', 'price': 2000}
```

## 4. Методы экземпляра

Метод — это функция, определённая внутри класса. Первый параметр — `self` (экземпляр). Методы могут изменять состояние экземпляра или возвращать что-то.

### Строковое представление: `__str__` и `__repr__`

- `__str__` — вызывается функциями `str()` и `print()`. Должен возвращать читаемую строку для пользователя.
- `__repr__` — вызывается функцией `repr()` и в интерактивной среде. Должен возвращать однозначное представление, часто похожее на конструктор.

```python
class Product:
    def __init__(self, name, price):
        self.name = name
        self.price = price

    def __str__(self):
        return f"Product(name={self.name!r}, price={self.price})"
        # !r вызывает repr(self.name)

    def __repr__(self):
        return f"Product('{self.name}', {self.price})"

laptop = Product("Laptop", 2000)
print(laptop)          # Product(name='Laptop', price=2000)  -> __str__
print(repr(laptop))    # Product('Laptop', 2000)            -> __repr__
```

### Пользовательский метод

```python
class Product:
    # ... (__init__, __str__)
    def make_discount(self, percent):
        """Применяет скидку в процентах, изменяет цену."""
        self.price = self.price * (100 - percent) / 100

laptop = Product("Laptop", 2000)
laptop.make_discount(10)   # скидка 10%
print(laptop)              # Product(name='Laptop', price=1800.0)
```

Обратите внимание: при вызове `laptop.make_discount(10)` в `self` автоматически подставляется `laptop`.

Можно также написать обычную функцию, принимающую объект:

```python
def apply_discount(product, percent):
    product.price = product.price * (100 - percent) / 100

apply_discount(laptop, 10)
print(laptop)   # цена станет 1620.0 (ещё одна скидка)
```

Но метод «привязан» к классу и логически относится именно к продуктам.

## 5. Методы класса (`@classmethod`) и статические методы (`@staticmethod`)

### Метод класса

- Декорируется `@classmethod`.
- Первый параметр — `cls` (сам класс, а не экземпляр).
- Может быть вызван как на классе, так и на экземпляре.
- Часто используется для альтернативных конструкторов.

```python
class Date:
    def __init__(self, year, month, day):
        self.year = year
        self.month = month
        self.day = day

    def __str__(self):
        return f"{self.__class__.__name__}(year={self.year}, month={self.month}, day={self.day})"

    @classmethod
    def from_string(cls, date_str):
        """Создаёт объект Date из строки вида '2023-11-25'."""
        year, month, day = map(int, date_str.split('-'))
        return cls(year, month, day)

# Использование
d1 = Date(2020, 5, 10)
d2 = Date.from_string("2023-11-25")
print(d1)   # Date(year=2020, month=5, day=10)
print(d2)   # Date(year=2023, month=11, day=25)
```

### Статический метод

- Декорируется `@staticmethod`.
- Не принимает ни `self`, ни `cls`. Это просто функция, которая находится в пространстве имён класса.
- Используется, когда логика относится к классу, но не зависит от его состояния.

```python
class Date:
    # ... (остальное)
    @staticmethod
    def is_valid_date_string(date_str):
        """Проверяет, что строка имеет формат YYYY-MM-DD и год, месяц, день в разумных пределах."""
        parts = date_str.split('-')
        if len(parts) != 3:
            return False
        try:
            y, m, d = map(int, parts)
        except ValueError:
            return False
        return 1 <= m <= 12 and 1 <= d <= 31 and 0 <= y <= 9999

print(Date.is_valid_date_string("2023-11-25"))   # True
print(Date.is_valid_date_string("2023-13-01"))   # False
```

**Почему `@classmethod` предпочтительнее `@staticmethod`?**
Если вы в будущем переименуете класс или создадите подкласс, `@classmethod` будет работать правильно (используя `cls`), а `@staticmethod` может быть жёстко привязан к имени класса. Однако статические методы полезны для вспомогательных функций, которые не требуют доступа ни к классу, ни к экземпляру.

## 6. Свойства (`@property`)

`@property` позволяет определить метод, который вызывается как атрибут (без скобок). Это полезно для вычисляемых значений или для контроля доступа (геттеры/сеттеры).

### Простой геттер

```python
class Square:
    def __init__(self, side):
        self.side = side

    @property
    def area(self):
        return self.side ** 2

sq = Square(5)
print(sq.area)   # 25 (вызывается без скобок)
# sq.area = 100  # AttributeError: can't set attribute
```

### Геттер и сеттер

Если нужно разрешить присваивание, добавляем сеттер:

```python
class Square:
    def __init__(self, side):
        self._side = side   # "защищённый" атрибут

    @property
    def side(self):
        return self._side

    @side.setter
    def side(self, value):
        if value <= 0:
            raise ValueError("Side must be positive")
        self._side = value

    @property
    def area(self):
        return self.side ** 2

sq = Square(5)
print(sq.side)    # 5
sq.side = 7
print(sq.area)    # 49
# sq.side = -2    # ValueError: Side must be positive
```

### Вычисляемое свойство, зависящее от другого

В видео был пример с квадратом, у которого `B` вычисляется как `A`:

```python
class Rectangle:
    def __init__(self, a, b):
        self.a = a
        self.b = b

    @property
    def area(self):
        return self.a * self.b

class Square(Rectangle):
    def __init__(self, a):
        super().__init__(a, a)   # вызываем инициализатор родителя
        # но тогда b не зависит от a при изменении a

# Чтобы b автоматически менялось при изменении a, нужно свойство:
class BetterSquare(Rectangle):
    def __init__(self, side):
        self._side = side
        super().__init__(self._side, self._side)  # пока что b = side

    @property
    def a(self):
        return self._side

    @a.setter
    def a(self, value):
        self._side = value
        self.b = value   # синхронизируем b

    @property
    def b(self):
        return self._side

    @b.setter
    def b(self, value):
        self._side = value
        self.a = value

sq = BetterSquare(5)
print(sq.a, sq.b)  # 5 5
sq.a = 7
print(sq.b)        # 7
```

Но проще в `Square` переопределить метод `area` и не использовать `b` вообще. В видео автор показал подход с `@property` для `b`, который возвращает `self.a`.

## 7. Наследование (одинарное и множественное)

### Одинарное наследование

Класс-наследник получает все методы и атрибуты родителя и может их переопределять.

```python
class Shape:
    def area(self):
        raise NotImplementedError

class Rectangle(Shape):
    def __init__(self, width, height):
        self.width = width
        self.height = height

    def area(self):
        return self.width * self.height

class Square(Rectangle):
    def __init__(self, side):
        super().__init__(side, side)  # вызываем родительский __init__

r = Rectangle(4, 5)
print(r.area())   # 20

s = Square(6)
print(s.area())   # 36
```

**Порядок разрешения методов (MRO):**
`Square.mro()` → `[Square, Rectangle, Shape, object]`.

### Вызов родительского метода через `super()`

`super()` возвращает специальный объект, который делегирует вызовы родительским классам. Это нужно, чтобы не повторять код и правильно работать в множественном наследовании.

```python
class Vehicle:
    def __init__(self, color):
        self.color = color

class Car(Vehicle):
    def __init__(self, color, wheels):
        super().__init__(color)   # вызываем родительский __init__
        self.wheels = wheels
```

### Множественное наследование

Класс может наследоваться от нескольких родителей. При поиске методов используется MRO (C3-линеаризация).

```python
class Flyer:
    def fly(self):
        print("Flying")

class Swimmer:
    def swim(self):
        print("Swimming")

class Duck(Flyer, Swimmer):
    pass

duck = Duck()
duck.fly()   # Flying
duck.swim()  # Swimming
```

**Проблема ромбовидного наследования** решается через MRO. Рассмотрим пример из видео:

```python
class Vehicle:
    def make_sound(self):
        print("No sound")

class Car(Vehicle):
    def make_sound(self):
        print("Beep")

class Ship(Vehicle):
    def make_sound(self):
        print("Woo")

class Amphibian(Car, Ship):
    pass

amph = Amphibian()
amph.make_sound()  # Beep (потому что Car идёт первым в списке наследования)
print(Amphibian.mro())
# [<class '__main__.Amphibian'>, <class '__main__.Car'>, <class '__main__.Ship'>, <class '__main__.Vehicle'>, <class 'object'>]
```

Если бы мы написали `class Amphibian(Ship, Car):`, то вывод был бы "Woo".

### Доступ к методам из разных родителей

Все методы обоих родителей доступны, если они уникальны:

```python
class Amphibian(Car, Ship):
    pass

amph = Amphibian("green", 4)  # если __init__ Car требует color и wheels
# но у Ship тоже есть свой __init__ (требует color, cabins). Кто будет вызван?
# Тот, кто первый в MRO (Car), если в Amphibian нет своего __init__.
# Чтобы использовать оба, нужно переопределить __init__ в Amphibian и вызывать обоих.
```

В видео автор создал класс `Amphibian` с собственным `__init__`, который инициализирует нужные атрибуты, а методы `make_sound`, `open_trunk`, `set_sail` работают благодаря наследованию.

## 8. Валидация атрибутов через `@classmethod` и `@property`

В видео показан пример, где на классе `Rectangle` определён метод класса `validate_value`, который проверяет, что значение — число. Затем этот метод используется в сеттерах свойств.

```python
class Rectangle:
    @classmethod
    def validate_value(cls, value):
        if not isinstance(value, (int, float)):
            raise TypeError(f"Value must be int or float, got {type(value).__name__}")
        return value

    def __init__(self, a, b):
        self._a = self.validate_value(a)
        self._b = self.validate_value(b)

    @property
    def a(self):
        return self._a

    @a.setter
    def a(self, value):
        self._a = self.validate_value(value)

    # аналогично для b

rect = Rectangle(4, 5)
# rect.a = "abc"   # TypeError: Value must be int or float, got str
```

Такой подход позволяет централизовать проверку.

## 9. Полный пример из видео (собранный и работающий)

Я объединю основные моменты в один код, который можно запустить и увидеть вывод.

```python

class Product:
    """Класс продукта с именем и ценой."""
    def __init__(self, name, price):
        self.name = name
        self.price = price

    def __str__(self):
        return f"Product(name={self.name!r}, price={self.price})"

    def __repr__(self):
        return f"Product('{self.name}', {self.price})"

        self.day = day

    def __str__(self):

    def from_string(cls, date_str):

    @staticmethod
    def is_valid_date_string(date_str):

        print("Generic vehicle sound")

class Car(Vehicle):
    def __init__(self, color, wheels):

    def make_sound(self):

    d = Date.from_string("2025-03-15")
    print(d)
    print(f"Is valid '2025-02-30'? {Date.is_valid_date_string('2025-02-30')}")
    print()

    print("=== Shape inheritance ===")
    rect = Rectangle(4, 5)
    sq = Square(6)
    print(f"Rectangle area: {rect.area()}")
    print(f"Square area: {sq.area()}")
    print()

    print("=== Multiple inheritance ===")
    amphib = Amphibian("green", 4)
    amphib.make_sound()   # Beep! (Car первый)
    amphib.open_trunk()   # от Car
    amphib.set_sail()     # от Ship
    amphib.toggle_mode()
```

## Итоги занятия

1. **Docstring** — строки документации для модулей, функций, классов. Доступны через `__doc__`.
2. **Классы** — чертежи для создания объектов. Экземпляры создаются вызовом класса.
3. **Атрибуты** могут быть на уровне класса (общие) или на уровне экземпляра (уникальные). Поиск атрибутов идёт от экземпляра к классу и выше.
4. **`__init__`** — инициализатор, вызывается автоматически. Задаёт начальные атрибуты.
5. **Методы экземпляра** — обычные функции в классе, первым параметром принимают `self`.
6. **`__str__`** и **`__repr__`** — строковые представления.
7. **`@classmethod`** — метод класса, принимает `cls`. Удобен для альтернативных конструкторов.
8. **`@staticmethod`** — статический метод, не принимает ни `self`, ни `cls`. Просто функция внутри класса.
9. **`@property`** — позволяет обращаться к методу как к атрибуту. Сеттер (`@attr.setter`) контролирует присваивание.
10. **Наследование** — позволяет переиспользовать код. `super()` вызывает родительские методы. MRO определяет порядок поиска.
11. **Множественное наследование** — класс может наследовать от нескольких родителей. Порядок родителей влияет на приоритет методов.

Теперь вы имеете твёрдую основу для дальнейшего изучения ООП в Python. На следующем занятии, вероятно, будут исключения и, возможно, больше про магические методы. Если что-то осталось непонятным — спрашивайте, разберём ещё детальнее!
