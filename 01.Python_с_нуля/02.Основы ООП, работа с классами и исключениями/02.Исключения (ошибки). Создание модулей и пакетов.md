## 1. Что такое исключения и зачем они нужны

**Исключение** — это событие, которое возникает при выполнении программы, когда что-то идёт не так (ошибка). Вместо того чтобы продолжать работу с неправильными данными, Python "бросает" (raise) исключение, и если его не обработать, программа остановится.

Пример: попытка возвести число в степень, где показатель — строка.

```python
# step1.py
a = 3
b = "2"   # строка, а не число

print(a ** b)
```

**Вывод (ошибка):**
```
TypeError: unsupported operand type(s) for ** or pow(): 'int' and 'str'
```

Программа прервалась. Если бы дальше был ещё код, он бы не выполнился.

## 2. Когда лучше выбросить исключение, а когда вернуть None

Представим функцию, которая возвращает погоду для города. Если город неизвестен, можно вернуть `None` или выбросить исключение. Исключение полезно, когда ситуация действительно ошибочная и дальнейшие вычисления невозможны.

```python
def get_weather_for(city):
    """Возвращает словарь с погодой, если city == 'Moscow', иначе бросает ValueError."""
    if city != "Moscow":
        raise ValueError(f"Invalid city: {city}")
    return {"temperature": 20, "humidity": 60, "rain_chance": 30}

def rain_tomorrow(city):
    """Возвращает True, если вероятность дождя > 50%, False если ≤ 50%, None если город неизвестен."""
    print(f"Will it rain tomorrow in {city}?")
    try:
        weather = get_weather_for(city)
    except ValueError:
        # Если город не поддерживается, возвращаем None (не знаем)
        return None
    return weather["rain_chance"] > 50

# Проверка
print(rain_tomorrow("Moscow"))   # нормально
print(rain_tomorrow("Sochi"))    # ошибка перехвачена, вернёт None
```

**Вывод:**
```
Will it rain tomorrow in Moscow?
False
Will it rain tomorrow in Sochi?
None
```

Обратите внимание: программа не упала, потому что мы перехватили исключение `ValueError` в `rain_tomorrow` и вернули `None`. Если бы мы не перехватили исключение, выполнение остановилось бы.

## 3. Конструкция `try` / `except` / `else` / `finally`

### 3.1 Базовый `try` / `except`

```python
def safe_divide(a, b):
    try:
        result = a / b
    except ZeroDivisionError:
        print("Ошибка: деление на ноль")
        return None
    return result

print(safe_divide(10, 2))   # 5.0
print(safe_divide(10, 0))   # Ошибка: деление на ноль → None
```

**Вывод:**
```
5.0
Ошибка: деление на ноль
None
```

### 3.2 Несколько `except` и получение объекта исключения

```python
def safe_divide(a, b):
    try:
        result = a / b
    except ZeroDivisionError:
        print("ZeroDivisionError: деление на ноль")
        return None
    except TypeError as e:
        print(f"TypeError: {e}")
        return None
    return result

print(safe_divide(10, "2"))   # TypeError: unsupported operand type(s) for /: 'int' and 'str'
```

**Вывод:**
```
TypeError: unsupported operand type(s) for /: 'int' and 'str'
None
```

### 3.3 Блок `else` (выполняется, если исключений не было)

```python
def safe_divide(a, b):
    try:
        c = a / b
    except ZeroDivisionError:
        print("На ноль делить нельзя")
        return None
    except TypeError:
        print("Неверный тип данных")
        return None
    else:
        print("Деление успешно выполнено")
        return c

print(safe_divide(8, 4))
```

**Вывод:**
```
Деление успешно выполнено
2.0
```

### 3.4 Блок `finally` (выполняется всегда, даже после `return`)

`finally` полезен для закрытия ресурсов (файлы, соединения с БД).

```python
def safe_divide(a, b):
    try:
        c = a / b
    except ZeroDivisionError:
        print("Ошибка: деление на ноль")
        return None
    else:
        print("Успешно")
        return c
    finally:
        print("Блок finally выполнен (очистка ресурсов)")

print(safe_divide(9, 3))
print(safe_divide(9, 0))
```

**Вывод:**
```
Успешно
Блок finally выполнен (очистка ресурсов)
3.0
Ошибка: деление на ноль
Блок finally выполнен (очистка ресурсов)
None
```

Обратите внимание: `finally` выполнился даже после `return`.

## 4. Создание собственных исключений

Исключения — это классы, наследующие от `Exception` (или от его наследников, например `ValueError`).

```python
# step3.py
class MyException(Exception):
    """Моё собственное исключение."""
    pass

# Можно создать экземпляр и выбросить
err = MyException("Что-то пошло не так")
print(repr(err))   # MyException('Что-то пошло не так')

# raise MyException("Ошибка в программе")  # раскомментируйте, чтобы увидеть
```

**Вывод:**
```
MyException('Что-то пошло не так')
```

Если раскомментировать `raise`, программа выведет:
```
Traceback (most recent call last):
  ...
__main__.MyException: Ошибка в программе
```

### Наследование от конкретного типа (например, `ValueError`)

```python
class InvalidOperandError(ValueError):
    """Исключение для неверного операнда."""
    pass
```

## 5. Пример: класс `Point` с собственным исключением и магическими методами

Создадим класс точки с координатами `x` и `y`, переопределим `__add__` и `__iadd__`, добавим проверку типа и своё исключение.

```python
# step4.py
class InvalidOperandError(ValueError):
    pass

class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __str__(self):
        return f"[{self.x}, {self.y}]"

    def __repr__(self):
        return f"Point(x={self.x}, y={self.y})"

    def __add__(self, other):
        """Вызывается при p1 + p2. Возвращает новую точку."""
        if not isinstance(other, Point):
            raise InvalidOperandError(f"Should be Point, not {type(other).__name__}")
        return Point(self.x + other.x, self.y + other.y)

    def __iadd__(self, other):
        """Вызывается при p1 += p2. Изменяет текущий объект."""
        if not isinstance(other, Point):
            raise InvalidOperandError(f"Should be Point, not {type(other).__name__}")
        self.x += other.x
        self.y += other.y
        return self   # обязательно вернуть self

p1 = Point(2, 3)
p2 = Point(5, 6)

p3 = p1 + p2
print(p3)          # [7, 9]
print(repr(p3))    # Point(x=7, y=9)

# Проверка +=
print(id(p1))      # какой-то адрес
p1 += p2
print(p1)          # [7, 9]
print(id(p1))      # тот же адрес (объект изменён, а не заменён)

# Попытка сложить с числом
# p4 = p1 + 10     # InvalidOperandError: Should be Point, not int
```

**Вывод (примерный):**
```
[7, 9]
Point(x=7, y=9)
140234567890123
[7, 9]
140234567890123
```

Если раскомментировать последнюю строку, получим:
```
InvalidOperandError: Should be Point, not int
```

## 6. Сокрытие имён (name mangling) и `@property`

В Python нет строгих приватных атрибутов, но есть соглашение: `_name` — "защищённый" (не трогай без нужды), а `__name` (два подчёркивания в начале) приводит к искажению имени: внутри класса доступно как `self.__name`, а снаружи — как `_ClassName__name`.

### Пример с паролем

```python
# step5.py
class User:
    def __init__(self, name, password):
        self.name = name
        self.__password = password   # искажённое имя

    @property
    def password(self):
        """Геттер — позволяет читать пароль."""
        return self.__password

    @password.setter
    def password(self, new_password):
        """Сеттер — позволяет установить пароль (можно добавить хеширование)."""
        print(f"Устанавливаем пароль: {new_password}")
        # Здесь можно захэшировать пароль
        self.__password = hash(new_password)   # для примера используем встроенный hash

user = User("John", "abc123")
print(user.name)          # John
print(user.password)      # через геттер получаем abc123

user.password = "secret"  # вызывается сеттер
print(user.password)      # теперь хеш от "secret"

# Прямой доступ к искажённому имени (не рекомендуется)
print(user._User__password)   # то же значение, что и user.password
```

**Вывод:**
```
John
abc123
Устанавливаем пароль: secret
-6094472904775304663   # (хеш, число)
-6094472904775304663
```

**Важно:** сокрытие имени не делает атрибут недоступным — опытный разработчик может обратиться к `_User__password`. Но это служит сигналом: "не лезь сюда".

## 7. Модули и пакеты

### 7.1 Что такое модуль и пакет

- **Модуль** — любой файл с расширением `.py`. Имя модуля — имя файла (без `.py`).
- **Пакет** — папка, содержащая файл `__init__.py` (может быть пустым). Позволяет группировать модули.

Структура проекта:
```
project/
│
├── main.py
├── modules/
│   ├── __init__.py
│   └── user.py
└── common.py
```

### 7.2 Импорт модулей и объектов

```python
# user.py
class User:
    def __init__(self, name):
        self.name = name

    def greet(self):
        return f"Hello, {self.name}!"

# __init__.py может быть пустым

# main.py
import modules.user           # импорт модуля
from modules.user import User # импорт класса напрямую
from common import my_var     # импорт переменной из другого модуля

u = User("Alice")
print(u.greet())
```

**Вывод:**
```
Hello, Alice!
```

### 7.3 Пространства имён

Каждый модуль имеет своё пространство имён. Это предотвращает конфликты имён.

### 7.4 `if __name__ == "__main__"`

Когда Python запускает файл как основную программу, переменная `__name__` в этом файле получает значение `"__main__"`. Если файл импортируется, `__name__` равно имени модуля. Это позволяет писать код, который выполняется только при прямом запуске, но не при импорте.

```python
# example.py
print(f"Модуль {__name__} загружен")

def useful_function():
    return 42

if __name__ == "__main__":
    print("Этот код выполняется только при запуске example.py, а не при импорте")
    print(useful_function())
```

**Запуск `python example.py`**:
```
Модуль __main__ загружен
Этот код выполняется только при запуске example.py, а не при импорте
42
```

**Импорт в другом файле**:
```python
import example   # выведет "Модуль example загружен", но не выполнит if-блок
```

## 8. Циклические импорты и как их избежать

Циклический импорт возникает, когда модуль A импортирует B, а B импортирует A (прямо или косвенно). Это может привести к ошибке.

**Плохо:**
```python
# main2.py
import main3   # циклическая зависимость
my_var = 25

# main3.py
from main2 import my_var   # ошибка: main2 ещё не полностью загружен
```

**Решение:** вынести общие данные в отдельный модуль.

```python
# common.py
my_var = 25
print(f"common loaded, __name__ = {__name__}")

# main2.py
from common import my_var
print(f"main2: {my_var}")

# main3.py
from common import my_var
import math
print(f"main3: sqrt({my_var}) = {math.sqrt(my_var)}")
```

Теперь циклической зависимости нет, модуль `common` загружается один раз, и оба используют одну переменную.

## 9. Полный пример: объединяем исключения и модули

Создадим пакет `shapes`, внутри — модуль `point.py` с нашим классом `Point` и исключением `InvalidOperandError`. В `main.py` импортируем и используем.

**Структура:**
```
shapes/
    __init__.py
    point.py
main.py
```

**shapes/point.py:**
```python
class InvalidOperandError(ValueError):
    pass

class Point:
    def __init__(self, x, y):
        self._x = x
        self._y = y

    @property
    def x(self):
        return self._x

    @property
    def y(self):
        return self._y

    def __add__(self, other):
        if not isinstance(other, Point):
            raise InvalidOperandError(f"Expected Point, got {type(other).__name__}")
        return Point(self.x + other.x, self.y + other.y)

    def __repr__(self):
        return f"Point({self.x}, {self.y})"
```

**main.py:**
```python
from shapes.point import Point, InvalidOperandError

p1 = Point(1, 2)
p2 = Point(3, 4)
p3 = p1 + p2
print(p3)   # Point(4, 6)

try:
    p4 = p1 + 100
except InvalidOperandError as e:
    print(f"Ошибка: {e}")
```

**Вывод:**
```
Point(4, 6)
Ошибка: Expected Point, got int
```

## Итоги занятия

1. **Исключения** — способ сообщить об ошибке. Используйте `raise` для генерации, `try`/`except` для перехвата.
2. Конструкция `try` может включать `except` (один или несколько), `else` (при отсутствии исключений), `finally` (выполняется всегда).
3. **Свои исключения** — классы, наследующие от `Exception` (или его потомков). Позволяют точно идентифицировать ошибки вашего кода.
4. **Name mangling** (два подчёркивания) — механизм для сокрытия атрибутов, но не абсолютной защиты. Используется вместе с `@property` для контролируемого доступа.
5. **Модули** — `.py`-файлы; **пакеты** — папки с `__init__.py`. Они организуют код и создают пространства имён.
6. `if __name__ == "__main__"` — защита от выполнения кода при импорте.
7. **Циклические импорты** решаются вынесением общих данных в отдельный модуль.
