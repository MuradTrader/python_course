## 1. Что такое HTTP? Клиент и сервер

**HTTP** (HyperText Transfer Protocol) — это правила, по которым клиент (твоя программа, браузер) и сервер обмениваются данными.
Обычно клиент **запрашивает** информацию, сервер **отвечает**.

В Python мы будем писать **клиентскую часть** — программу, которая отправляет запросы и получает ответы.

### Установка библиотеки requests

Открой терминал (командную строку) и выполни:

```bash
pip install requests
```

Если у тебя несколько версий Python, возможно, нужно `pip3`. Убедись, что установка прошла успешно.

## 2. Первый GET-запрос. Статус-коды

**GET** — метод для получения данных. Например, ты открываешь страницу в браузере — браузер шлёт GET-запрос.

Создай файл `http_examples.py` и напишем первый код:

```python
import requests

# Отправляем GET-запрос на тестовый сервис httpbin.org/get
response = requests.get("https://httpbin.org/get")

print("Статус-код:", response.status_code)
print("Тип ответа:", type(response))
print("Содержимое ответа (текст):", response.text[:200])  # первые 200 символов
```

**Вывод в консоли:**

```
Статус-код: 200
Тип ответа: <class 'requests.models.Response'>
Содержимое ответа (текст): {
  "args": {},
  "headers": {
    "Accept": "*/*",
    "Accept-Encoding": "gzip, deflate, br",
    "Host": "httpbin.org",
    "User-Agent": "python-requests/2.31.0",
    "X-Amzn-Trace-Id": "Root=...
```

**Разбор:**
- `requests.get(url)` — отправляет GET-запрос и возвращает объект `Response`.
- `response.status_code` — код ответа сервера. **200** означает «OK, всё хорошо».
- `response.text` — тело ответа в виде строки (обычно HTML, JSON, XML). Здесь сервер вернул JSON.

### Группы статус-кодов (как в видео)

| Диапазон | Группа | Примеры |
|----------|--------|----------|
| 1xx | Информационные | 100 Continue |
| 2xx | Успешные | **200 OK**, 201 Created, 204 No Content |
| 3xx | Перенаправления | 301 Moved Permanently, 302 Found |
| 4xx | Ошибка клиента | **404 Not Found**, 400 Bad Request, 401 Unauthorized, 403 Forbidden |
| 5xx | Ошибка сервера | 500 Internal Server Error, 502 Bad Gateway |

Проверим несколько статус-кодов на практике:

```python
import requests

urls = [
    "https://httpbin.org/status/200",
    "https://httpbin.org/status/404",
    "https://httpbin.org/status/500",
    "https://httpbin.org/status/418"   # "Я чайник" (шутка)
]

for url in urls:
    resp = requests.get(url)
    print(f"{url} -> {resp.status_code} {resp.reason}")
```

**Вывод:**

```
https://httpbin.org/status/200 -> 200 OK
https://httpbin.org/status/404 -> 404 NOT FOUND
https://httpbin.org/status/500 -> 500 INTERNAL SERVER ERROR
https://httpbin.org/status/418 -> 418 I'M A TEAPOT
```

- `resp.reason` — текстовое пояснение к коду.
- Статус **418 I'm a teapot** — реальный код из RFC 2324, прикол для кофеварок.

**Зачем это нужно?** Твоя программа может проверить `status_code` и решить, что делать дальше: обработать данные, повторить запрос или показать ошибку.

## 3. Методы HTTP: GET, POST, PUT, PATCH, DELETE

Методы (глаголы) указывают, какое действие мы хотим выполнить над ресурсом.

| Метод | Назначение | Безопасный? | Идемпотентный? | Тело запроса |
|-------|------------|-------------|----------------|---------------|
| GET | Получить данные | Да | Да | Не имеет |
| POST | Создать новый ресурс | Нет | Нет | Есть |
| PUT | Полностью заменить ресурс | Нет | Да | Есть |
| PATCH | Частично изменить ресурс | Нет | Нет | Есть |
| DELETE | Удалить ресурс | Нет | Да | Обычно нет |

**Безопасный** — не изменяет состояние сервера (только чтение).
**Идемпотентный** — повторный одинаковый запрос даёт тот же результат (например, удалить один и тот же товар дважды — товара уже нет, состояние не меняется).

### 3.1 GET (мы уже делали)

Можно передавать параметры в адресной строке (query string):

```python
import requests

params = {"q": "python requests", "page": 2}
response = requests.get("https://httpbin.org/get", params=params)

print("URL, на который реально ушёл запрос:", response.url)
print("Параметры, которые увидел сервер:", response.json()["args"])
```

**Вывод:**

```
URL, на который реально ушёл запрос: https://httpbin.org/get?q=python+requests&page=2
Параметры, которые увидел сервер: {'q': 'python requests', 'page': '2'}
```

- `params` автоматически кодирует словарь в строку запроса.
- Сервер (httpbin) возвращает нам эти параметры в поле `args`.

### 3.2 POST — создание ресурса

POST обычно отправляет данные в **теле запроса**. Это могут быть данные формы, JSON, файлы.

**Пример отправки JSON:**

```python
import requests

new_data = {
    "name": "Harry Potter",
    "year": 1997,
    "author": "J.K. Rowling"
}

response = requests.post("https://httpbin.org/post", json=new_data)

print("Статус:", response.status_code)
print("Ответ сервера (JSON):")
print(response.json()["json"])   # сервер эхом возвращает отправленный JSON
```

**Вывод:**

```
Статус: 200
Ответ сервера (JSON):
{'name': 'Harry Potter', 'year': 1997, 'author': 'J.K. Rowling'}
```

- `json=new_data` автоматически устанавливает заголовок `Content-Type: application/json` и сериализует словарь в JSON.
- Если бы мы слали форму (как в видео), использовали бы `data={"username": "john", "password": "123"}`.

**Что такое тело запроса?** Это дополнительные данные, которые не помещаются в URL. Например, пароль или большой текст.

### 3.3 PUT и PATCH — изменение

```python
import requests

# PUT — замена всего объекта
updated_book = {
    "name": "Harry Potter and the Chamber of Secrets",
    "year": 1998,
    "author": "J.K. Rowling"
}
resp_put = requests.put("https://httpbin.org/put", json=updated_book)
print("PUT:", resp_put.status_code)

# PATCH — частичное обновление (только year)
patch_data = {"year": 1999}
resp_patch = requests.patch("https://httpbin.org/patch", json=patch_data)
print("PATCH:", resp_patch.status_code)
```

**Вывод:**

```
PUT: 200
PATCH: 200
```

httpbin всегда отвечает 200 на такие запросы (это тестовый сервер). В реальном API ты бы получил 200 или 204.

### 3.4 DELETE — удаление

```python
response = requests.delete("https://httpbin.org/delete")
print(response.status_code)
# 200 (httpbin просто имитирует удаление)
```

## 4. Заголовки (Headers)

Заголовки — это мета-информация о запросе или ответе: тип содержимого, длина, авторизация, user-agent и т.д. В Python их можно отправлять через параметр `headers`.

**Пример: отправляем кастомный заголовок**

```python
import requests

my_headers = {
    "X-My-Custom-Header": "Hello from Python",
    "User-Agent": "MySuperApp/1.0"
}

response = requests.get("https://httpbin.org/headers", headers=my_headers)
data = response.json()
print("Заголовки, которые увидел сервер:")
print(data["headers"])
```

**Вывод (часть):**

```
Заголовки, которые увидел сервер:
{
  "X-My-Custom-Header": "Hello from Python",
  "User-Agent": "MySuperApp/1.0",
  "Host": "httpbin.org",
  ...
}
```

**Чтение заголовков ответа сервера:**

```python
resp = requests.get("https://httpbin.org/get")
print("Content-Type:", resp.headers.get("Content-Type"))
print("Все заголовки:", dict(resp.headers))
```

Вывод будет содержать `content-type: application/json` и другие служебные поля.

## 5. Параметры строки запроса (Query String)

Это часть URL после знака `?`. Мы уже использовали `params` в GET. Вот ещё пример с несколькими одинаковыми ключами (сервер увидит список):

```python
params = {"foo": ["bar", "baz"]}   # два значения для foo
resp = requests.get("https://httpbin.org/get", params=params)
print(resp.url)
print(resp.json()["args"])
```

**Вывод:**

```
https://httpbin.org/get?foo=bar&foo=baz
{'foo': ['bar', 'baz']}
```

## 6. Тело запроса: form-data vs JSON

### 6.1 Отправка формы (application/x-www-form-urlencoded)

```python
form_data = {"username": "alice", "password": "secret"}
resp = requests.post("https://httpbin.org/post", data=form_data)
print(resp.json()["form"])   # сервер вернёт эти данные в поле "form"
```

**Вывод:** `{'username': 'alice', 'password': 'secret'}`

### 6.2 Отправка JSON (мы уже видели)

```python
resp = requests.post("https://httpbin.org/post", json={"key": "value"})
print(resp.json()["json"])   # {'key': 'value'}
```

Разница только в параметре: `data` для формы, `json` для JSON. Сервер определяет формат по заголовку `Content-Type`.

## 7. Тело ответа: текст, JSON, бинарные данные

Ответ сервера может быть в любом формате. `requests` предоставляет удобные методы:

```python
resp = requests.get("https://httpbin.org/json")
print("Как текст:", resp.text[:100])
print("Как JSON:", resp.json())        # автоматически парсит JSON
print("Как байты:", resp.content[:20]) # для картинок, файлов
```

**Вывод (пример для JSON):**

```
Как текст: {
  "slideshow": {
    "author": "Yours Truly",
    "date": "date of publication",
...
Как JSON: {'slideshow': {'author': 'Yours Truly', ...}}
Как байты: b'{\n  "slideshow": {\n    "author": "Yours Truly",\n    ...
```

- Если сервер вернул HTML, XML, картинку — используй `resp.text` или `resp.content`.
- `resp.json()` вызовет исключение, если ответ не в формате JSON.

## 8. Редиректы (3xx)

Когда сервер возвращает код 301, 302 и заголовок `Location`, клиент должен перейти по новому адресу. `requests` делает это **автоматически** по умолчанию.

```python
resp = requests.get("https://httpbin.org/redirect/1")  # один редирект
print("Конечный статус:", resp.status_code)
print("История редиректов:", [r.status_code for r in resp.history])
```

**Вывод:**

```
Конечный статус: 200
История редиректов: [302]
```

- `resp.history` — список промежуточных ответов.
- Если не хочешь следовать редиректам, укажи `allow_redirects=False`.

## 9. Обработка ошибок и статус-кодов

Всегда проверяй статус-код, прежде чем обрабатывать данные.

```python
resp = requests.get("https://httpbin.org/status/404")
if resp.status_code == 200:
    print("Успех!")
elif resp.status_code == 404:
    print("Ресурс не найден. Проверь URL.")
else:
    print(f"Ошибка {resp.status_code}: {resp.reason}")
```

Можно использовать `raise_for_status()` — он выбросит исключение для 4xx/5xx:

```python
resp = requests.get("https://httpbin.org/status/404")
try:
    resp.raise_for_status()
    print("Всё хорошо")
except requests.exceptions.HTTPError as e:
    print(f"Ошибка HTTP: {e}")
```

**Вывод:** `Ошибка HTTP: 404 Client Error: NOT FOUND for url: ...`

## 10. REST API на практике (небольшой проект)

REST API — это способ построения веб-сервисов, который использует HTTP методы и статусы по назначению. Например, типичные endpoint'ы:

- `GET /books` — получить список книг
- `GET /books/1` — получить книгу с id=1
- `POST /books` — создать книгу (данные в теле)
- `PUT /books/1` — полностью обновить книгу 1
- `PATCH /books/1` — частично обновить
- `DELETE /books/1` — удалить книгу

Мы можем протестировать это на тестовом API `jsonplaceholder.typicode.com` (фейковые данные).

```python
import requests

# GET список постов
response = requests.get("https://jsonplaceholder.typicode.com/posts")
print("Всего постов:", len(response.json()))
print("Первый пост:", response.json()[0]["title"])

# POST создать новый пост
new_post = {
    "title": "Мой первый пост",
    "body": "Содержимое поста",
    "userId": 1
}
resp = requests.post("https://jsonplaceholder.typicode.com/posts", json=new_post)
print("Создан пост с id:", resp.json()["id"])   # сервер вернёт 201 и присвоит id=101
```

**Вывод (примерный):**

```
Всего постов: 100
Первый пост: sunt aut facere repellat provident occaecati excepturi optio reprehenderit
Создан пост с id: 101
```
